## Docusaurus 究竟是什麼？

Docusaurus 是一款**靜態網站生成器**，能協助您**快速建立**美觀的**文件網站**。

專注於您的內容：只需撰寫**Markdown文件**。Docusaurus 將為您生成經過優化的**網站**，並可輕鬆**部署至任何地方**。

Docusaurus 具備**完整功能**且極度**靈活**：我們提供精心設計的文件與部落格版面，開箱即用的版本控制、搜尋與國際化功能，同時兼顧無障礙存取與搜尋引擎優化。其彈性的主題系統可讓您**調整UI以符合品牌形象**，完美整合至主網站或文件入口。採用**React**技術實現**現代化客戶端導航**，並能打造**互動式文件**。

![Slash介紹圖](/img/slash-introducing.svg)

Docusaurus 的哲學符合**帕累托法則**：您只需投入**20%的精力**即可獲得**80%的成果**。這讓您能用**最小成本**打造可媲美頂尖水準的文件網站。

## Docusaurus 的誕生故事

Docusaurus 由**Facebook開源團隊**（現為[Meta開源](https://opensource.fb.com/)）於**2017年**創建。當時我們有大量內部與開源專案需要文件化。**撰寫優質文件已屬不易**，更遑論打造美觀網站所需的HTML、CSS與JavaScript。我們希望專案負責人能**專注於內容**，而**Markdown**正是最佳選擇。

當時的解決方案是**不斷複製/貼上Jekyll模板**。這自然導致**維護困難**，因此我們開發了這個工具來**一勞永逸解決痛點**。

**[Docusaurus v1 誕生了](/blog/2017/12-14-introducing-docusaurus.mdx)!**

![Slash 的誕生](/img/slash-birth.png)

它迅速在 Facebook 和前端生態系統中獲得動力，被許多知名專案採用，例如 [Prettier](https://prettier.io/)、[Babel](https://babeljs.io/)、[React-Native](https://archive.reactnative.dev/)、[KaTeX](https://katex.org/docs/)，當然還有 [Docusaurus v1](http://v1.docusaurus.io/) 本身。

## 邁向 Docusaurus 2.0

[**Docusaurus v1**](http://v1.docusaurus.io/) 非常成功，但我們開始**質疑一些架構上的選擇**：

- React 僅作為**伺服器端模板語言**使用，並未在客戶端使用
- **主題系統相當有限**，除了用 CSS 更改一些顏色外，很難進行更高級的自訂
- **文件版本控制系統令人困惑**，因為它基於差異演算法
- 程式碼庫是**單一的**，既沒有經過良好測試，也不易擴展

[**Docusaurus v2**](https://docusaurus.io/) 從頭開始**重建**，採用了新的**模組化架構**：

- React 現在也在客戶端使用，實現了**現代單頁應用程式導航**
- **外掛**讓社群能夠以第三方套件的形式貢獻有用的功能
- **主題**比以往更加**靈活**
- 文件版本控制現在基於快照副本，更容易理解
- 我們保留了**v1 的所有優點**：文件、部落格、頁面、版本控制、國際化...
- 我們實現了**多項新功能**

更多細節請參閱 [Docusaurus 2 專案公告](/blog/2018/09-11-Towards-Docusaurus-2.mdx) 和 [v1 到 v2 遷移指南](https://docusaurus.io/docs/migration)

## 2.0 版本有哪些新功能？

很難一一列舉 Docusaurus v2 的所有新功能，我們將重點介紹最具影響力的功能。

### MDX

[MDX](https://github.com/mdx-js/mdx) 允許您在 Markdown 中**嵌入 React 組件**，這讓您能輕鬆打造頂尖的**互動式文檔體驗**。

實際示範勝過千言萬語：

```md title="docs/my-document.mdx"
### Give it a try: press that button!

import ColorModeToggle from '@theme/ColorModeToggle';

<ColorModeToggle/>
```

```mdx-code-block
<BrowserWindow>

<h3>Give it a try: press that button!</h3>

<ColorModeToggle/>

</BrowserWindow>
```

:::info

MDX 擁有自己的[插件系統](https://mdxjs.com/docs/extending-mdx/)，您可以自訂 Markdown 撰寫體驗，甚至創建自己的 Markdown 語法。

:::

### 主題系統

主題系統是 Docusaurus 最重要的功能之一：我們認為專業的文檔網站應該**符合您公司的品牌形象**，創造一致的體驗。

Docusaurus 的主題系統在多重層級上提供了極大的**靈活性**：

- 自訂 CSS 變數來調整顏色、字體等
- 提供您自己的 CSS 樣式表
- 從頭實作自己的主題
- **覆寫預設主題的任何 React 組件**：我們稱之為 [swizzling](https://docusaurus.io/docs/swizzling)

## 為何現在推出 2.0？

許多熱心追蹤者好奇**為何我們花了 4 年才發布 Docusaurus 2.0**，尤其測試版已相當成功且**廣泛用於生產環境**。

原因是我們堅持**遵循[語意化版本控制](https://semver.org/)**，這意味著每當發布**重大變更**時，我們就會增加主版本號。

這點至關重要：

- 只要您僅使用[公開 API](/community/release-process#public-api-surface)，就能**保證次要版本升級簡單**
- 符合前端生態系慣例
- 主版本更新是徹底記錄重大變更的契機
- 主/次版本更新可透過部落格文章宣傳新功能

問題在於我們彈性的主題系統本質上形成了非常**隱性的 API 介面**，難以判斷何謂重大變更。高度客製化的 Docusaurus 網站有時升級困難，因為它們使用內部 API 實現客製化。我們投入時間進行主題重構並明確定義[公開 API](/community/release-process#public-api-surface)，使未來程式變更更安全。我們將持續擴展此公開主題 API，讓常見的網站客製化無需使用任何內部 API。

:::info

從現在起，Docusaurus 將**更頻繁發布主版本**。實際上，您可預期**每 2 至 4 個月就會有新主版本**。

[主版本號並非神聖不可侵犯](https://tom.preston-werner.com/2022/05/23/major-version-numbers-are-not-sacred.html)，但我們仍會將重大變更集中處理，避免過於頻繁發布主版本。

詳情請參閱我們的[發布流程](/community/release-process)文件。

:::
