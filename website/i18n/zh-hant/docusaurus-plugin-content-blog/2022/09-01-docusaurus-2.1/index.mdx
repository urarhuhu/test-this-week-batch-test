我們很高興宣布 **Docusaurus 2.1**，這是我們首個次要版本更新。

升級過程應該很簡單：正如我們的[發佈流程文件](/community/release-process)所述，次要版本遵循[語意化版本控制](https://semver.org/)規範。

## 重點更新

### DocCardList 改進

在 [#8008](https://github.com/facebook/docusaurus/pull/8008) 中，我們簡化了 `<DocCardList>` 元件的使用方式，該元件主要用於側邊欄分類的生成索引頁面。

`items` 屬性現在是選填的，會自動從父級側邊欄分類的內容推斷：

```diff
import DocCardList from '@theme/DocCardList';
- import {useCurrentSidebarCategory} from '@docusaurus/theme-common';

- <DocCardList items={useCurrentSidebarCategory().items}/>
+ <DocCardList/>
```

此外，我們讓它能在任何文件上使用，包括未連結到任何側邊欄分類的常規文件。

![簡化的 DocCardList 元件](./img/doc-card-list.png)

### `noindex` 改進

我們加強了對 [`noindex` 元 `robots` 指令](https://developers.google.com/search/docs/advanced/crawling/block-indexing)的支援，該指令用於向搜尋引擎表明您不希望特定頁面被索引。

在 [#7963](https://github.com/facebook/docusaurus/pull/7963) 中，我們允許按文件版本配置 `noindex`。

使用以下插件選項告訴爬蟲您不希望 `1.0.0` 版本被索引：

```jsx
const options = {
  versions: {
    '1.0.0': {
      noIndex: true,
    },
  },
};
```

實際上，Docusaurus 會為該版本的每個頁面添加以下元標記：

```html
<meta name="robots" content="noindex, nofollow" />
```

在 [#7964](https://github.com/facebook/docusaurus/pull/7964) 中，我們也修復了一個錯誤，即網站地圖插件仍會包含帶有 `noindex` 指令的頁面。現在網站地圖插件會可靠地過濾掉所有包含 `noindex` 指令的頁面。

### 覆蓋預設元標記

在 [#7952](https://github.com/facebook/docusaurus/pull/7952) 中，現在可以覆蓋以前無法覆蓋的預設 HTML 元標記：

```jsx
<>
  {siteConfig.noIndex && <meta name="robots" content="nofollow, noindex" />}
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
</>
```

現在可以使用 `<Head>` 或 `themeConfig.metadata`：

- 覆蓋 `viewport` 元標記
- 覆蓋 `robots` 元標記：您可以將整個網站標記為 `noIndex`，但特定頁面除外，這些頁面仍應被索引
