# 介紹

⚡️ Docusaurus 能幫助您快速建立一個**美觀的文件網站**。

💸 自建技術堆疊成本高昂。相反地，**專注於您的內容**，只需撰寫 Markdown 檔案即可。

💥 準備好進階功能了嗎？使用**進階功能**如版本控制、國際化、搜尋和主題客製化。

💅 查看**[最佳 Docusaurus 網站](/showcase?tags=favorite)**獲取靈感，並閱讀一些**[用戶評價](https://x.com/sebastienlorber/timelines/1392048416872706049)**。

🧐 Docusaurus 是一個**靜態網站生成器**。它建構了一個**單頁應用程式**，具有快速的客戶端導航，充分利用**React**的強大功能使您的網站互動性更強。它提供了開箱即用的**文件功能**，但也可用於建立**任何類型的網站**（個人網站、產品、部落格、行銷登陸頁面等）。

![](/img/slash-introducing.svg)

## 快速入門 ⏱️ {#fast-track}

透過實際操作，**5分鐘**內了解 Docusaurus！

建立一個新的 Docusaurus 網站並遵循**非常簡短**的內嵌教學。

安裝 [Node.js](https://nodejs.org/en/download/) 並建立一個新的 Docusaurus 網站：

```bash
npx create-docusaurus@latest my-website classic
```

啟動網站：

```bash
cd my-website
npx docusaurus start
```

開啟 [`http://localhost:3000`](http://localhost:3000) 並遵循教學。

:::tip

使用 **[docusaurus.new](https://docusaurus.new)** 立即在瀏覽器中測試 Docusaurus！

或線上閱讀 **[5分鐘教學](https://tutorial.docusaurus.io)**。

:::

## 從 v1 遷移 {#migrating-from-v1}

Docusaurus v2+ 是對 Docusaurus v1 的完全重寫，利用了完全現代化的工具鏈。在 [v2 正式發布](https://docusaurus.io/blog/2022/08/01/announcing-docusaurus-2.0) 後，我們強烈建議您**使用 Docusaurus v2+ 而非 Docusaurus v1**，因為 Docusaurus v1 已被棄用。

已有[許多用戶](/showcase)在使用 Docusaurus v2+（[趨勢](https://www.npmtrends.com/docusaurus-vs-@docusaurus/core)）。

**在以下情況下使用 Docusaurus v2+：**

- :white_check_mark: 您想要一個現代的 Jamstack 文件網站
- :white_check_mark: 您想要一個具有客戶端路由的單頁應用程式 (SPA)
- :white_check_mark: 您想要 React 和 MDX 的完整功能
- :white_check_mark: 您不需要支援 IE11

**在以下情況下使用 [Docusaurus v1](https://v1.docusaurus.io/)：**

- :x: 您不想要單頁應用程式 (SPA)
- :x: 您需要支援 IE11（...真的需要嗎？IE [已停止支援](https://docs.microsoft.com/en-us/lifecycle/products/internet-explorer-11)）

對於現有的 v1 用戶，若想升級至 v2+，可以遵循我們的[遷移指南](./migration/index.mdx)。

## 功能特色 {#features}

Docusaurus 的設計高度重視開發者與貢獻者的使用體驗。

- ⚛️ **以 React 與 💚 打造**：
  - 可透過 React 進行擴展與客製化
  - 透過提供自訂 React 元件，全面掌控網站的瀏覽體驗
- **模組化設計**：
  - 使用基礎模板快速建立網站，再透過進階功能與插件強化
  - 可開源插件與社群共享
- ✂️ **開發者體驗**：
  - 立即開始撰寫文件
  - 通用配置入口點，便於貢獻者維護
  - 變更時支援熱重載與閃電般快速的增量建置
  - 基於路由的程式碼與資料分割
  - 輕鬆部署至 GitHub Pages、Netlify、Vercel 等服務

我們的共同目標——協助使用者快速找到所需內容，更深入理解您的產品。我們分享最佳實踐，助您打造完善的文件網站。

- 🎯 **SEO 友善**：
  - 為所有可能路徑靜態生成 HTML 檔案
  - 頁面專屬 SEO 優化，讓使用者直接找到與問題相關的官方文件
- 📝 **基於 MDX**：
  - 透過嵌入 Markdown 的 JSX 與 React 撰寫互動元件
  - 透過即時編輯器分享程式碼，讓使用者當場愛上您的產品
- 🔍 **搜尋功能**：全站內容皆可搜尋
- 💾 **文件版本控制**：保持文件與專案版本同步
- 🌍 **國際化 (i18n)**：支援多語言網站翻譯

Docusaurus v2+ 天生具備極致效能，並貼心照顧所有使用者的可訪問性需求。

- ⚡️ **閃電般快速**。Docusaurus v2+ 遵循 [PRPL 模式](https://developers.google.com/web/fundamentals/performance/prpl-pattern/)，確保內容載入速度極快。
- 🦖 **無障礙設計**。注重可訪問性，讓所有使用者都能平等使用您的網站。

## 設計原則 {#design-principles}

- **低學習曲線**。Docusaurus API 簡潔易學易用。即使需要更多程式碼與時間，使用者仍能實現大多數功能。與其設計錯誤的抽象層，不如不抽象，我們不希望使用者被迫繞過錯誤的抽象設計。必看演講—[最小化 API 表面積](https://www.youtube.com/watch?v=4anAwXYqLG8)。
- **直覺化**。檢視 Docusaurus 專案目錄或新增功能時，使用者不會感到無所適從。架構應直覺易懂，能基於熟悉的方式輕鬆擴充。
- **分層架構**。堆疊各層級（內容/主題/樣式）之間的關注點分離必須清晰——良好抽象且模組化。
- **合理預設值**。為使用者預設實施常見的效能優化與配置，同時保留覆寫選項。
- **無廠商鎖定**。不強制使用預設插件或 CSS（但強烈建議）。核心基礎設施如 React Loadable 和 React Router 因預設效能優化而不可替換，但高階元件可自由選擇。Markdown 引擎、CSS 框架、CSS 方法論等架構完全由使用者決定。

我們相信，作為開發者，了解工具運作原理能提升使用技巧。因此我們致力於解說 Docusaurus 的架構與各元件，希望使用者能深入理解工具，更熟練地運用它。

## 與其他工具比較 {#comparison-with-other-tools}

在所有靜態網站生成器中，Docusaurus 獨特聚焦於文件網站，並提供眾多開箱即用功能。

我們亦研究其他主流靜態網站生成器，願分享比較見解，協助您在眾多選擇中找到方向。

### Gatsby {#gatsby}

[Gatsby](https://www.gatsbyjs.com/) 具備豐富功能，擁有龐大的插件生態系統，能實現 Docusaurus 的所有功能。但這自然伴隨著較高的學習曲線。Gatsby 在多方面表現出色，適合構建各種類型的網站。而 Docusaurus 則專注於將一件事做到極致——成為編寫與發布內容的最佳工具。

GraphQL 在 Gatsby 中佔據核心地位，儘管構建 Gatsby 網站不一定需要使用它。在大多數靜態網站開發場景中，您並不需要 GraphQL 提供的靈活性。

Docusaurus v2+ 的許多設計靈感源自 Gatsby 的優點，它是一個出色的替代方案。

[Docz](https://github.com/pedronauck/docz) 是基於 Gatsby 的主題，用於構建文檔網站。目前其功能完整度低於 Docusaurus。

### Next.js {#nextjs}

[Next.js](https://nextjs.org/) 是另一個熱門的混合式 React 框架。它雖能協助構建文檔網站，但並未針對文檔使用場景進行預設優化，需要投入更多工作量才能實現 Docusaurus 的開箱即用功能。

[Nextra](https://github.com/shuding/nextra) 是基於 Next.js 的靜態網站生成器，具有明確的設計導向。目前其功能完整度低於 Docusaurus。

### VuePress {#vuepress}

[VuePress](https://vuepress.vuejs.org/) 與 Docusaurus 有諸多相似之處——兩者都專注於內容型網站，並提供開箱即用的文檔功能。但 VuePress 基於 Vue 構建，而 Docusaurus 基於 React。若您需要 Vue 技術棧的解決方案，VuePress 會是不錯的選擇。

### MkDocs {#mkdocs}

[MkDocs](https://www.mkdocs.org/) 是流行的 Python 靜態網站生成器，其核心價值主張與 Docusaurus 相似。

如果您不需要單頁應用且不計劃使用 React，這會是個理想的選擇。

[Material for MkDocs](https://squidfunk.github.io/mkdocs-material/) 提供了精美的主題方案。

### Docsify {#docsify}

[Docsify](https://docsify.js.org/) 能快速創建文檔網站，但它並非靜態網站生成器，且對 SEO 不友好。

### GitBook {#gitbook}

[GitBook](https://www.gitbook.com/) 擁有簡潔的設計風格，曾被眾多開源項目採用。隨著其產品定位逐漸轉向商業化而非開源工具，許多功能已不符合開源項目文檔站點的需求，促使大量用戶轉向其他方案。您可參閱 [Redux 遷移至 Docusaurus 的討論](https://github.com/reduxjs/redux/issues/3161)了解詳情。

目前 GitBook 僅對開源團隊和非營利組織免費，而 Docusaurus 對所有用戶免費開放。

### Jekyll {#jekyll}

[Jekyll](https://github.com/jekyll/jekyll) 是最成熟的靜態網站生成器之一——事實上在 Docusaurus 出現前，Facebook 多數開源網站都基於 Jekyll 構建！它的入門門檻極低。我們希望為用戶提供與 Jekyll 相似的開發體驗。

相較於通過 `<script />` 標籤為靜態 HTML 添加交互性的傳統方案，Docusaurus 網站本質是 React 應用。我們希望通過現代 JavaScript 生態工具鏈，為文檔站點的性能標準、資源構建流程、優化方案及易用性樹立新標杆。
