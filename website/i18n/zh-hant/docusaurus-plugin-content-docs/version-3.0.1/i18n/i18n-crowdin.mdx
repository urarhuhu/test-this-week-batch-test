# 國際化 - 使用 Crowdin

Docusaurus 的國際化系統**與任何翻譯軟體解耦**。

您可以將 Docusaurus 與**您選擇的工具和 SaaS 服務**整合，只要將**翻譯文件放在正確的位置**即可。

我們以 [Crowdin](https://crowdin.com/) 的使用為例，作為**一種**可能的**整合範例**。

:::warning

這**並非對 Crowdin 作為唯一選擇的背書**，但 Facebook 已成功使用它來翻譯如 [Jest](https://jestjs.io/)、[Docusaurus](https://docusaurus.io/) 和 [ReasonML](https://reasonml.github.io/) 等文件專案。

如需協助，請參考 **[Crowdin 文件](https://support.crowdin.com/)** 和 **[Crowdin 支援](mailto:support@crowdin.com)**。

:::

:::tip

使用這個 **[社群驅動的 GitHub 討論](https://github.com/facebook/docusaurus/discussions/4052)** 來討論任何與 Docusaurus + Crowdin 相關的事項。

:::

## Crowdin 概述 {#crowdin-overview}

Crowdin 是一個翻譯 SaaS 服務，為開源專案提供[免費方案](https://crowdin.com/page/open-source-project-setup-request)。

我們推薦以下翻譯工作流程：

- 將**原始檔案上傳**至 Crowdin（未翻譯的檔案）
- 使用 Crowdin **翻譯內容**
- 從 Crowdin **下載翻譯**（本地化的翻譯檔案）

Crowdin 提供了一個 [CLI](https://support.crowdin.com/cli-tool/) 來**上傳原始檔案**和**下載翻譯**，讓您可以自動化翻譯流程。

[`crowdin.yml` 設定檔](https://support.crowdin.com/configuration-file/) 對 Docusaurus 很方便，並允許**將本地化的翻譯檔案下載到預期的位置**（在 `i18n/[locale]/..` 中）。

閱讀**[官方文件](https://support.crowdin.com/)**以了解更多進階功能和不同的翻譯工作流程。

## Crowdin 教學 {#crowdin-tutorial}

這是一個使用 Crowdin 將新初始化的英文 Docusaurus 網站翻譯成法文的逐步指南，假設您已經遵循了[國際化教學](./i18n-tutorial.mdx)。

最終結果可以在 [docusaurus-crowdin-example.netlify.app](https://docusaurus-crowdin-example.netlify.app/) 查看（[儲存庫](https://github.com/slorber/docusaurus-crowdin-example)）。

### 準備 Docusaurus 網站 {#prepare-the-docusaurus-site}

初始化一個新的 Docusaurus 網站：

```bash
npx create-docusaurus@latest website classic
```

為法文語言新增網站設定：

```js title="docusaurus.config.js"
export default {
  i18n: {
    defaultLocale: 'en',
    locales: ['en', 'fr'],
  },
  themeConfig: {
    navbar: {
      items: [
        // ...
        {
          type: 'localeDropdown',
          position: 'left',
        },
        // ...
      ],
    },
  },
  // ...
};
```

翻譯首頁：

```jsx title="src/pages/index.js"
import React from 'react';
import Translate from '@docusaurus/Translate';
import Layout from '@theme/Layout';

export default function Home() {
  return (
    <Layout>
      <h1 style={{margin: 20}}>
        <Translate description="The homepage main heading">
          Welcome to my Docusaurus translated site!
        </Translate>
      </h1>
    </Layout>
  );
}
```

### 建立 Crowdin 專案 {#create-a-crowdin-project}

在 [Crowdin](https://crowdin.com/) 上註冊並建立一個專案。

使用英文作為原始語言，法文作為目標語言。

![建立一個 Crowdin 專案，以英文為原始語言，法文為目標語言](/img/crowdin/crowdin-create-project.png)

您的專案已建立，但目前是空的。我們將在接下來的步驟中上傳要翻譯的檔案。

### 建立 Crowdin 設定 {#create-the-crowdin-configuration}

此設定（[文件](https://support.crowdin.com/configuration-file/)）為 Crowdin CLI 提供了一個映射，以便理解：

- Where to find the source files to upload (JSON and Markdown)
- Where to download the files after translation (in `i18n/[locale]`)

Create `crowdin.yml` in `website`:

```yml title="crowdin.yml"
project_id: '123456'
api_token_env: CROWDIN_PERSONAL_TOKEN
preserve_hierarchy: true
files:
  # JSON translation files
  - source: /i18n/en/**/*
    translation: /i18n/%two_letters_code%/**/%original_file_name%
  # Docs Markdown files
  - source: /docs/**/*
    translation: /i18n/%two_letters_code%/docusaurus-plugin-content-docs/current/**/%original_file_name%
  # Blog Markdown files
  - source: /blog/**/*
    translation: /i18n/%two_letters_code%/docusaurus-plugin-content-blog/**/%original_file_name%
```

Crowdin has its own syntax for declaring source/translation paths:

- `**/*`: everything in a subfolder
- `%two_letters_code%`: the 2-letters variant of Crowdin target languages (`fr` in our case)
- `**/%original_file_name%`: the translations will preserve the original folder/file hierarchy

:::info

The Crowdin CLI warnings are not always easy to understand.

We advise to:

- change one thing at a time
- re-upload sources after any configuration change
- use paths starting with `/` (`./` does not work)
- avoid fancy globbing patterns like `/docs/**/*.(md|mdx)` (does not work)

:::

#### Access token {#access-token}

The `api_token_env` attribute defines the **env variable name** read by the Crowdin CLI.

You can obtain a `Personal Access Token` on [your personal profile page](https://crowdin.com/settings#api-key).

:::tip

You can keep the default value `CROWDIN_PERSONAL_TOKEN`, and set this environment variable and on your computer and on the CI server to the generated access token.

:::

:::warning

A Personal Access Tokens grant **read-write access to all your Crowdin projects**.

You should **not commit** it, and it may be a good idea to create a dedicated **Crowdin profile for your company** instead of using a personal account.

:::

#### Other configuration fields {#other-configuration-fields}

- `project_id`: can be hardcoded, and is found on `https://crowdin.com/project/<MY_PROJECT_NAME>/settings#api`
- `preserve_hierarchy`: preserve the folder's hierarchy of your docs on Crowdin UI instead of flattening everything

### Install the Crowdin CLI {#install-the-crowdin-cli}

This tutorial uses the CLI version `3.5.2`, but we expect `3.x` releases to keep working.

Install the Crowdin CLI as an npm package to your Docusaurus site:

```bash npm2yarn
npm install @crowdin/cli@3
```

Add a `crowdin` script:

```json title="package.json"
{
  "scripts": {
    // ...
    "write-translations": "docusaurus write-translations",
    "crowdin": "crowdin"
  }
}
```

Test that you can run the Crowdin CLI:

```bash npm2yarn
npm run crowdin -- --version
```

Set the `CROWDIN_PERSONAL_TOKEN` env variable on your computer, to allow the CLI to authenticate with the Crowdin API.

:::tip

Temporarily, you can hardcode your personal token in `crowdin.yml` with `api_token: 'MY-TOKEN'`.

:::

### Upload the sources {#upload-the-sources}

Generate the JSON translation files for the default language in `website/i18n/en`:

```bash npm2yarn
npm run write-translations
```

Upload all the JSON and Markdown translation files:

```bash npm2yarn
npm run crowdin upload
```

![Crowdin CLI uploading Docusaurus source files](/img/crowdin/crowdin-upload-sources-cli.png)

Your source files are now visible on the Crowdin interface: `https://crowdin.com/project/<MY_PROJECT_NAME>/settings#files`

![Crowdin UI showing Docusaurus source files](/img/crowdin/crowdin-source-files.png)

### Translate the sources {#translate-the-sources}

On `https://crowdin.com/project/<MY_PROJECT_NAME>`, click on the French target language.

![Crowdin 介面顯示法文翻譯檔案](/img/crowdin/crowdin-french-translations.png)

翻譯部分 Markdown 檔案。

![Crowdin 介面翻譯 Markdown 檔案](/img/crowdin/crowdin-translate-markdown.png)

:::tip

使用 `Hide String` 功能確保翻譯人員**不會翻譯不應翻譯的內容**：

- Front matter：`id`、`slug`、`tags`...
- Admonitions：`:::`、`:::note`、`:::tip`...

![Crowdin 介面隱藏字串](/img/crowdin/crowdin-hide-string.png)

:::

翻譯部分 JSON 檔案。

![Crowdin 介面翻譯 JSON 檔案](/img/crowdin/crowdin-translate-json.png)

:::info

JSON 翻譯檔案的 `description` 屬性會在 Crowdin 上顯示，協助翻譯字串。

:::

:::tip

**[預先翻譯](https://support.crowdin.com/pre-translation-via-machine/)** 你的網站，並**手動修正預先翻譯的錯誤**（請先在設定中啟用 Global Translation Memory）。

建議先使用 `Hide String` 功能，因為 Crowdin 的預先翻譯有時會過於樂觀。

:::

### 下載翻譯檔案 {#download-the-translations}

使用 Crowdin CLI 下載已翻譯的 JSON 和 Markdown 檔案。

```bash npm2yarn
npm run crowdin download
```

翻譯內容應會下載至 `i18n/fr` 目錄。

以法語語系啟動你的網站：

```bash npm2yarn
npm run start -- --locale fr
```

確認你的網站現在已翻譯為法語，網址為 [`http://localhost:3000/fr/`](http://localhost:3000/fr/)。

### 透過 CI 自動化 {#automate-with-ci}

我們將設定 CI **在建置時下載 Crowdin 翻譯檔案**，並將其保留在 Git 之外。

將 `website/i18n` 加入 `.gitignore`。

在你的 CI 中設定 `CROWDIN_PERSONAL_TOKEN` 環境變數。

建立一個 npm 腳本來 `sync` Crowdin（提取來源檔案、上傳來源檔案、下載翻譯檔案）：

```json title="package.json"
{
  "scripts": {
    "crowdin:sync": "docusaurus write-translations && crowdin upload && crowdin download"
  }
}
```

在你的 CI 中呼叫 `npm run crowdin:sync` 腳本，就在建置 Docusaurus 網站之前。

:::tip

保持部署預覽的速度：不要下載翻譯檔案，並對功能分支使用 `npm run build -- --locale en`。

:::

:::warning

Crowdin 不支援多個同時進行的上傳/下載：建議僅在生產部署中包含翻譯檔案，並保持部署預覽未翻譯狀態。

:::

## Crowdin 進階主題 {#advanced-crowdin-topics}

### MDX {#mdx}

:::warning

特別注意 MDX 文件中的 JSX 片段！

:::

Crowdin **官方不支援 MDX**，但他們已新增**對 `.mdx` 副檔名的支援**，並將此類檔案視為 Markdown（而非純文字）。

#### MDX 問題 {#mdx-problems}

Crowdin 會將 JSX 語法視為嵌入式 HTML，並可能在翻譯檔案下載時干擾 JSX 標記，導致網站因無效 JSX 而無法建置。

使用簡單字串屬性的簡單 JSX 片段（如 `<Username name="Sebastien"/>`）可以正常運作；但使用物件/陣列屬性的複雜 JSX 片段（如 `<User person={{name: "Sebastien"}}/>`）則更容易因語法不像 HTML 而失敗。

#### MDX 解決方案 {#mdx-solutions}

我們建議將複雜的嵌入式 JSX 代碼提取為獨立的組件。同時我們也新增了 `mdx-code-block` 逃逸語法：

`````text
# How to deploy Docusaurus

To deploy Docusaurus, run the following command:

````mdx-code-block 

import Tabs from '@theme/Tabs';

import TabItem from '@theme/TabItem';

<Tabs>
  <TabItem value="bash" label="Bash">

  ```bash
  GIT_USER=<GITHUB_USERNAME> yarn deploy
  ```

  </TabItem>
  <TabItem value="windows" label="Windows">

  ```batch
  cmd /C "set "GIT_USER=<GITHUB_USERNAME>" && yarn deploy"
  ```

  </TabItem>
</Tabs>
````
`````

這將實現以下效果：

- 被 Crowdin 識別為代碼塊（下載翻譯時不會破壞標記結構）
- 被 Docusaurus 識別為常規 JSX（就像未被任何代碼塊包裹一樣）
- 但會失去 MDX 工具支援（IDE 語法高亮、Prettier 格式化等）

### 文檔版本控制 {#docs-versioning}

為 `website/versioned_docs` 目錄配置翻譯文件。

當創建新版本時，源字符串通常與當前版本 (`website/docs`) 高度相似，此時您不希望反覆重新翻譯新版本文檔。

Crowdin 提供「重複字符串」設定選項。

![Crowdin 重複字符串設定選項](/img/crowdin/crowdin-settings-duplicate-strings.png)

建議選擇「隱藏」模式，但最佳設定取決於各版本間的差異程度。

:::warning

若未使用「隱藏」模式，會大幅增加配額中的「源字符串」數量，進而影響計費。

:::

### 多實例插件 {#multi-instance-plugins}

需為每個插件實例單獨配置翻譯文件。

若您擁有 `id=ios` 的文檔插件實例，則需同時配置以下源文件：

- `website/ios`
- `website/ios_versioned_docs` (若啟用版本控制)

### 站點維護 {#maintaining-your-site}

當您在 Git 上刪除或重命名源文件時，Crowdin CLI 會顯示警告：

![Crowdin CLI 下載翻譯警告](/img/crowdin/crowdin-download-translations-warning.png)

重構源文件後，應通過 Crowdin 界面手動更新文件：

![Crowdin 界面文件重命名操作](/img/crowdin/crowdin-files-rename.png)

### 版本控制系統集成 {#vcs-git-integrations}

Crowdin 提供多種 VCS 集成方案，包括 [GitHub](https://support.crowdin.com/github-integration/)、GitLab 和 Bitbucket。

:::warning[簡要說明]

建議避免使用這些集成方案。

:::

理論上這能實現 Git 與 Crowdin 的雙向編輯，並建立兩系統間的同步機制。

但實際運作中存在可靠性問題：

- Crowdin → Git 同步正常（通過拉取請求實現）
- Git → Crowdin 需手動觸發同步（需點擊按鈕）
- Crowdin 匹配現有 Markdown 翻譯與源文件的啟發式算法並非 100% 可靠，同步後需在界面人工驗證
- 用戶在 Git 和 Crowdin 並發編輯可能導致翻譯丟失
- 要求 `crowdin.yml` 文件必須位於倉庫根目錄

### 上下文本地化 {#in-context-localization}

Crowdin 提供[上下文本地化](https://support.crowdin.com/in-context-localization/)功能。

:::warning

目前因技術限制無法正常運作，但我們認為未來可能解決此問題。

Crowdin 會將 Markdown 字符串替換為技術性 ID（如 `crowdin:id12345`），但替換過程過於激進，會影響隱藏字符串、前言數據、提示框及 JSX 等結構。

:::

### 本地化編輯網址 {#localize-edit-urls}

當使用者瀏覽 `/fr/doc1` 頁面時，預設情況下編輯按鈕會連結到未本地化的文件 `website/docs/doc1.md`。

您可能更希望編輯按鈕連結到 Crowdin 介面，這時可以使用 `editUrl` 函數來根據不同語言環境自訂編輯網址。

```js title="docusaurus.config.js"
const DefaultLocale = 'en';

export default {
  presets: [
    [
      '@docusaurus/preset-classic',
      {
        docs: {
          // highlight-start
          editUrl: ({locale, versionDocsDirPath, docPath}) => {
            // Link to Crowdin for French docs
            if (locale !== DefaultLocale) {
              return `https://crowdin.com/project/docusaurus-v2/${locale}`;
            }
            // Link to GitHub for English docs
            return `https://github.com/facebook/docusaurus/edit/main/website/${versionDocsDirPath}/${docPath}`;
          },
          // highlight-end
        },
        blog: {
          // highlight-start
          editUrl: ({locale, blogDirPath, blogPath}) => {
            if (locale !== DefaultLocale) {
              return `https://crowdin.com/project/docusaurus-v2/${locale}`;
            }
            return `https://github.com/facebook/docusaurus/edit/main/website/${blogDirPath}/${blogPath}`;
          },
          // highlight-start
        },
      },
    ],
  ],
};
```

:::note

目前**無法連結到 Crowdin 中的特定檔案**。

:::

### 範例設定 {#example-configuration}

**Docusaurus 設定檔**是使用版本控制與多實例的良好範例：

```mdx-code-block
import CrowdinConfigV2 from '!!raw-loader!@site/../crowdin-v2.yaml';
import CodeBlock from '@theme/CodeBlock';

<CodeBlock className="language-yaml" title="crowdin.yml">
  {CrowdinConfigV2.split('\n')
    // remove comments
    .map((line) => !line.startsWith('#') && line)
    .filter(Boolean)
    .join('\n')}
</CodeBlock>
```
