# 建立頁面

本章節將介紹如何在 Docusaurus 中建立頁面。

`@docusaurus/plugin-content-pages` 插件讓您能建立**獨立單頁**，例如展示頁、遊樂場頁面或支援頁面。您可以使用 React 元件或 Markdown 來建立。

:::note

頁面沒有側邊欄，只有[文件](./docs/docs-introduction.mdx)才有。

:::

:::info

詳盡的選項列表請參閱[頁面插件 API 參考文件](./../api/plugins/plugin-content-pages.mdx)。

:::

## 新增 React 頁面 {#add-a-react-page}

使用 React 作為建立頁面的 UI 函式庫。每個頁面元件都應導出一個 React 元件，您可充分運用 React 的表現力來建立豐富的互動內容。

建立檔案 `/src/pages/helloReact.js`：

```jsx title="/src/pages/helloReact.js"
import React from 'react';
import Layout from '@theme/Layout';

export default function Hello() {
  return (
    <Layout title="Hello" description="Hello React Page">
      <div
        style={{
          display: 'flex',
          justifyContent: 'center',
          alignItems: 'center',
          height: '50vh',
          fontSize: '20px',
        }}>
        <p>
          Edit <code>pages/helloReact.js</code> and save to reload.
        </p>
      </div>
    </Layout>
  );
}
```

儲存檔案後，開發伺服器會自動重新載入變更。現在開啟 [`http://localhost:3000/helloReact`](http://localhost:3000/helloReact) 即可看到您剛建立的新頁面。

每個頁面預設不包含任何樣式。若需要顯示導覽列和/或頁尾，您需從 `@theme/Layout` 匯入 `Layout` 元件，並將內容包裹在該元件中。

:::tip

您也可使用 `.tsx` 副檔名建立 TypeScript 頁面 (`helloReact.tsx`)。

:::

## 新增 Markdown 頁面 {#add-a-markdown-page}

建立檔案 `/src/pages/helloMarkdown.md`：

```md title="/src/pages/helloMarkdown.md"
---
title: my hello page title
description: my hello page description
hide_table_of_contents: true
---

# Hello

How are you?
```

同樣地，系統會在 [`http://localhost:3000/helloMarkdown`](http://localhost:3000/helloMarkdown) 建立頁面。

Markdown 頁面的靈活性低於 React 頁面，因其始終使用主題佈局。

這裡有個 [Markdown 頁面範例](/examples/markdownPageExample)。

:::tip

您也可在 Markdown 頁面中充分運用 React 功能，詳見 [MDX](https://mdxjs.com/) 文件。

:::

## 路由 {#routing}

若您熟悉 Jekyll 或 Next 等靜態網站生成器，會對這種路由方式感到熟悉。在 `/src/pages/` 目錄下建立的任何 JavaScript 檔案，都會依循該目錄層級自動轉換為網站頁面。例如：

- `/src/pages/index.js` → `[baseUrl]`
- `/src/pages/foo.js` → `[baseUrl]/foo`
- `/src/pages/foo/test.js` → `[baseUrl]/foo/test`
- `/src/pages/foo/index.js` → `[baseUrl]/foo/`

在這個元件化開發時代，建議將樣式、標記和行為集中封裝在元件中。每個頁面都是一個元件，若需用自訂樣式設計頁面，建議將樣式與頁面元件集中存放在同目錄下。例如要建立「支援」頁面，您可選擇：

- 新增 `/src/pages/support.js` 檔案
- 建立 `/src/pages/support/` 目錄和 `/src/pages/support/index.js` 檔案

後者更為理想，因其能將頁面相關檔案集中存放。例如專用於「支援」頁面的 CSS 模組檔案 (`styles.module.css`)。

:::note

這僅是建議的目錄結構，您仍需手動在元件模組 (`support/index.js`) 中匯入 CSS 模組檔案。

:::

預設情況下，任何以 `_` 開頭的 Markdown 或 JavaScript 檔案都會被忽略，且不會為該檔案建立路由（請參閱 `exclude` 選項）。

```bash
my-website
├── src
│   └── pages
│       ├── styles.module.css
│       ├── index.js
│       ├── _ignored.js
│       ├── _ignored-folder
│       │   ├── Component1.js
│       │   └── Component2.js
│       └── support
│           ├── index.js
│           └── styles.module.css
.
```

:::warning

`src/pages/` 目錄下的所有 JavaScript/TypeScript 檔案都會自動生成對應的網站路徑。若需在該目錄建立可重複使用的元件，請使用 `exclude` 選項（預設情況下，以 `_` 開頭的檔案、測試檔案(`.test.js`) 及 `__tests__` 目錄中的檔案不會被轉換為頁面）。

:::

### 重複路由 {#duplicate-routes}

您可能會意外建立多個指向相同路由的頁面。發生此情況時，執行 `yarn start` 或 `yarn build` 時 Docusaurus 會發出重複路由警告，但網站仍會成功建置。最後建立的頁面將可存取，但會覆蓋其他衝突頁面。解決此問題需修改或移除衝突路由。
