## 語法高亮 {#syntax-highlighting}

程式碼區塊是由三個反引號包裹的文字區塊。您可查閱 [MDX 規範](https://github.com/mdx-js/specification) 了解詳細標準。

````md
```js
console.log('Every repo must come with a mascot.');
```
````

為程式碼區塊使用對應的語言標識，Docusaurus 將自動啟用由 [Prism React Renderer](https://github.com/FormidableLabs/prism-react-renderer) 提供的語法高亮功能。

### 主題設定 {#theming}

預設採用 Prism 的 [Palenight](https://github.com/FormidableLabs/prism-react-renderer/blob/master/packages/prism-react-renderer/src/themes/palenight.ts) [語法高亮主題](https://github.com/FormidableLabs/prism-react-renderer#theming)。您可通過在 docusaurus.config.js 的 `themeConfig` 中配置 `prism` 的 `theme` 字段來切換主題。

例如，若需改用 `dracula` 高亮主題：

```js title="docusaurus.config.js"
import {themes as prismThemes} from 'prism-react-renderer';

export default {
  themeConfig: {
    prism: {
      // highlight-next-line
      theme: prismThemes.dracula,
    },
  },
};
```

由於 Prism 主題本質是 JS 物件，您也可以自行編寫主題。Docusaurus 增強了 `github` 和 `vsDark` 主題以提供更豐富的高亮效果，可參考我們對 [淺色](https://github.com/facebook/docusaurus/blob/main/website/src/utils/prismLight.ts) 與 [深色](https://github.com/facebook/docusaurus/blob/main/website/src/utils/prismDark.ts) 程式碼區塊主題的實作。

### 支援語言 {#supported-languages}

Docusaurus 預設內建 [常用語言子集](https://github.com/FormidableLabs/prism-react-renderer/blob/master/packages/generate-prism-languages/index.ts#L9-L23)。

:::warning

部分流行語言如 Java、C# 或 PHP 預設未啟用。

:::

若要為其他 [Prism 支援語言](https://prismjs.com/#supported-languages) 添加語法高亮，需在附加語言陣列中定義。

:::note

每個附加語言必須是有效的 Prism 元件名稱。例如 Prism 會將語言標識 `cs` 映射到 `csharp`，但只有 `prism-csharp.js` 作為元件存在，因此需使用 `additionalLanguages: ['csharp']`。您可查閱 `node_modules/prismjs/components` 目錄獲取所有可用元件（語言）。

:::

例如，若要為 PowerShell 語言添加高亮支援：

```js title="docusaurus.config.js"
export default {
  // ...
  themeConfig: {
    prism: {
      // highlight-next-line
      additionalLanguages: ['powershell'],
    },
    // ...
  },
};
```

添加 `additionalLanguages` 後，需重啟 Docusaurus。

若要為 Prism 尚未支援的語言添加高亮，可替換 `prism-include-languages` 元件：

```bash npm2yarn
npm run swizzle @docusaurus/theme-classic prism-include-languages
```

此操作將在 `src/theme` 目錄生成 `prism-include-languages.js` 文件。通過編輯該文件可添加自訂語言的高亮支援：

```js title="src/theme/prism-include-languages.js"
const prismIncludeLanguages = (Prism) => {
  // ...

  additionalLanguages.forEach((lang) => {
    require(`prismjs/components/prism-${lang}`);
  });

  // highlight-next-line
  require('/path/to/your/prism-language-definition');

  // ...
};
```

編寫自訂語言定義時，可參考 [Prism 官方語言定義](https://github.com/PrismJS/prism/tree/master/components)。

當新增自訂語言定義時，您無需將該語言加入 `additionalLanguages` 設定陣列，因為 Docusaurus 僅會在 Prism 提供的語言中查詢 `additionalLanguages` 字串。在 `prism-include-languages.js` 中匯入語言即已足夠。

### 使用註解標記 {#highlighting-with-comments}

您可以使用帶有 `highlight-next-line`、`highlight-start` 和 `highlight-end` 的註解來選擇要標記的行數。

````md
```js
function HighlightSomeText(highlight) {
  if (highlight) {
    // highlight-next-line
    return 'This text is highlighted!';
  }

  return 'Nothing highlighted';
}

function HighlightMoreText(highlight) {
  // highlight-start
  if (highlight) {
    return 'This range is highlighted!';
  }
  // highlight-end

  return 'Nothing highlighted';
}
```
````

支援的註解語法：

| Style      | Syntax                   |
| ---------- | ------------------------ |
| C-style    | `/* ... */` and `// ...` |
| JSX-style  | `{/* ... */}`            |
| Bash-style | `# ...`                  |
| HTML-style | `<!-- ... -->`           |

我們會盡力根據語言推斷應使用哪組註解樣式，預設允許使用所有註解樣式。如果有目前不支援的註解樣式，我們樂意新增！歡迎提交 Pull Request。請注意，不同的註解樣式沒有語意差異，僅內容有別。

您可以在 `src/css/custom.css` 中為標記的程式碼行設定自訂背景顏色，以更符合您選擇的語法標記主題。下方提供的顏色適用於預設標記主題 (Palenight)，若您使用其他主題，則需相應調整顏色。

```css title="/src/css/custom.css"
:root {
  --docusaurus-highlighted-code-line-bg: rgb(72, 77, 91);
}

/* If you have a different syntax highlighting theme for dark mode. */
[data-theme='dark'] {
  /* Color which works with dark mode syntax highlighting theme */
  --docusaurus-highlighted-code-line-bg: rgb(100, 100, 100);
}
```

若您還需要以其他方式樣式化標記的程式碼行，可以針對 `theme-code-block-highlighted-line` CSS 類別進行設定。

### 自訂魔法註解 {#custom-magic-comments}

`// highlight-next-line` 和 `// highlight-start` 等被稱為「魔法註解」，因為它們會被解析並移除，其目的是為下一行或註解對包圍的區段新增元資料。

您可以透過主題設定宣告自訂魔法註解。例如，您可以註冊另一個會新增 `code-block-error-line` 類別名稱的魔法註解：

```mdx-code-block
<Tabs>
<TabItem value="docusaurus.config.js">
```

```js
export default {
  themeConfig: {
    prism: {
      magicComments: [
        // Remember to extend the default highlight class name as well!
        {
          className: 'theme-code-block-highlighted-line',
          line: 'highlight-next-line',
          block: {start: 'highlight-start', end: 'highlight-end'},
        },
        // highlight-start
        {
          className: 'code-block-error-line',
          line: 'This will error',
        },
        // highlight-end
      ],
    },
  },
};
```

若在元字串中使用數字範圍語法（如 `{1,3-4}`），Docusaurus 會套用 **第一個 `magicComments` 條目** 的類別名稱。預設情況下，此類別名稱為 `theme-code-block-highlighted-line`，但若您變更 `magicComments` 配置並使用不同的條目作為第一個，元字串範圍的意義也會隨之改變。

您可透過設定 `magicComments: []` 來停用預設的行高亮註解。若無任何魔法註解配置，但 Docusaurus 遇到包含元字串範圍的程式碼區塊時，將會報錯——因為沒有類別名稱可供套用（畢竟高亮類別名稱僅是魔法註解條目之一）。

每個魔法註解條目皆包含三個鍵值：必填的 `className`、適用於緊接下一行的 `line`，或包含 `start` 與 `end` 的 `block`（用於標註被兩個註解包圍的整個區塊）。

透過 CSS 鎖定類別已能實現許多效果，但您可透過 [swizzling](../../swizzling.mdx) 徹底釋放此功能的潛力。

```bash npm2yarn
npm run swizzle @docusaurus/theme-classic CodeBlock/Line
```

`Line` 元件會接收類別名稱清單，您可據此條件式渲染不同的標記。

## 互動式程式碼編輯器 {#interactive-code-editor}

（由 [React Live](https://github.com/FormidableLabs/react-live) 驅動）

您可使用 `@docusaurus/theme-live-codeblock` 外掛建立互動式程式碼編輯器。首先，將此外掛加入您的套件。

```bash npm2yarn
npm install --save @docusaurus/theme-live-codeblock
```

您還需在 `docusaurus.config.js` 中加入此外掛。

```js {3}
export default {
  // ...
  themes: ['@docusaurus/theme-live-codeblock'],
  // ...
};
```

使用此外掛時，請在語言元字串後附加 `live` 來建立程式碼區塊。

````md
```jsx live
function Clock(props) {
  const [date, setDate] = useState(new Date());
  useEffect(() => {
    const timerID = setInterval(() => tick(), 1000);

    return function cleanup() {
      clearInterval(timerID);
    };
  });

  function tick() {
    setDate(new Date());
  }

  return (
    <div>
      <h2>It is {date.toLocaleTimeString()}.</h2>
    </div>
  );
}
```
````

程式碼區塊將被渲染為互動式編輯器，程式碼變更會即時反映在結果面板上。

### 命令式渲染 (noInline)

當您的程式碼橫跨多個元件或變數時，應使用 `noInline` 選項以避免錯誤。

````md
```jsx live noInline
````

## 在程式碼區塊中使用 JSX 標記 {#using-jsx-markup}

Markdown 中的程式碼區塊始終將其內容保留為純文字，這意味著您無法實現如下操作：

```ts
type EditUrlFunction = (params: {
  // This doesn't turn into a link (for good reason!)
  version: <a href="/docs/versioning">Version</a>;
  versionDocsDirPath: string;
  docPath: string;
  permalink: string;
  locale: string;
}) => string | undefined;
```

若想嵌入如錨點連結或粗體等HTML標記，可使用`<pre>`標籤、`<code>`標籤或`<CodeBlock>`元件。

```jsx
<pre>
  <b>Input: </b>1 2 3 4{'\n'}
  <b>Output: </b>"366300745"{'\n'}
</pre>
```

<BrowserWindow>
  <pre>
    <b>{'Input: '}</b>
    {'1 2 3 4\n'}
    <b>{'Output: '}</b>
    {'"366300745"\n'}
  </pre>
</BrowserWindow>

:::warning[MDX對空白不敏感]

MDX遵循JSX行為：即使位於`<pre>`內，換行符也會被轉為空格。必須明確寫入換行字符才能實際輸出。

:::

:::warning

語法突顯僅作用於純文字字串。Docusaurus不會嘗試解析含有JSX子元素的程式碼區塊內容。

:::

### Docusaurus npm2yarn備註插件 {#npm2yarn-remark-plugin}

同時顯示npm與Yarn的CLI指令是常見需求，例如：

```bash npm2yarn
npm install @docusaurus/remark-plugin-npm2yarn
```

Docusaurus內建此功能，免去每次使用`Tabs`元件的麻煩。啟用方式：先安裝`@docusaurus/remark-plugin-npm2yarn`套件，接著在`docusaurus.config.js`中，於需要此功能的插件(doc、blog、pages等)的`remarkPlugins`選項註冊。（詳細配置格式請參閱[文檔配置](../../api/plugins/plugin-content-docs.mdx#ex-config)）

```js title="docusaurus.config.js"
export default {
  // ...
  presets: [
    [
      '@docusaurus/preset-classic',
      {
        docs: {
          // highlight-start
          remarkPlugins: [
            [require('@docusaurus/remark-plugin-npm2yarn'), {sync: true}],
          ],
          // highlight-end
        },
        pages: {
          // highlight-next-line
          remarkPlugins: [require('@docusaurus/remark-plugin-npm2yarn')],
        },
        blog: {
          // highlight-start
          remarkPlugins: [
            [
              require('@docusaurus/remark-plugin-npm2yarn'),
              {converters: ['pnpm']},
            ],
          ],
          // highlight-end
          // ...
        },
      },
    ],
  ],
};
```

使用時在程式碼區塊添加`npm2yarn`鍵：

````md
```bash npm2yarn
npm install @docusaurus/remark-plugin-npm2yarn
```
````

#### 配置選項 {#npm2yarn-remark-plugin-configuration}

| Option | Type | Default | Description |
| --- | --- | --- | --- |
| `sync` | `boolean` | `false` | Whether to sync the selected converter across all code blocks. |
| `converters` | `array` | `'yarn'`, `'pnpm'` | The list of converters to use. The order of the converters is important, as the first converter will be used as the default choice. |

## 在JSX中使用 {#usage-in-jsx}

在Markdown之外，可使用`@theme/CodeBlock`元件獲得相同輸出效果。

```jsx
import CodeBlock from '@theme/CodeBlock';

export default function MyReactPage() {
  return (
    <div>
      {/* highlight-start */}
      <CodeBlock
        language="jsx"
        title="/src/components/HelloCodeTitle.js"
        showLineNumbers>
        {`function HelloCodeTitle(props) {
  return <h1>Hello, {props.name}</h1>;
}`}
      </CodeBlock>
      {/* highlight-end */}
    </div>
  );
}
```

```mdx-code-block
<BrowserWindow>
  <CodeBlock
    language="jsx"
    title="/src/components/HelloCodeTitle.js"
    showLineNumbers>
    {`function HelloCodeTitle(props) {
  return <h1>Hello, {props.name}</h1>;
}`}
  </CodeBlock>
</BrowserWindow>
```

接受的屬性包括`language`、`title`與`showLineNumbers`，用法與Markdown程式碼區塊相同。

雖然不建議，但也可傳遞`metastring`屬性如`metastring='{1-2} title="/src/components/HelloCodeTitle.js" showLineNumbers'`，此為Markdown程式碼區塊的底層處理方式。但建議改為[使用註解突顯行數](#highlighting-with-comments)。

如[先前所述](#using-jsx-markup)，語法突顯僅在子元素為純文字字串時才會生效。
