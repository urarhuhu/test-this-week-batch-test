# 介紹

⚡️ Docusaurus 能協助您快速打造出**精美的文件網站**。

💸 自建技術堆疊成本高昂。請**專注於內容創作**，只需撰寫 Markdown 檔案即可。

💥 需要更多功能？可使用**進階特性**如版本控制、國際化、搜尋及主題客製化。

💅 查看**[精選 Docusaurus 網站](/showcase?tags=favorite)**獲取靈感，並閱讀**[用戶見證](https://x.com/sebastienlorber/timelines/1392048416872706049)**。

🧐 Docusaurus 是**靜態網站生成器**。它建構出具有快速客戶端導航的**單頁應用程式**，充分發揮 **React** 的互動能力。除預設的**文件功能**外，也能用於建立**任何類型網站**（個人網站、產品頁、部落格、行銷登陸頁等）。

![](/img/slash-introducing.svg)

## 快速入門 ⏱️ {#fast-track}

透過實作，**5分鐘**快速理解 Docusaurus！

新建 Docusaurus 網站並跟隨**極簡**內建教學。

安裝 [Node.js](https://nodejs.org/en/download/) 後建立新網站：

```bash
npx create-docusaurus@latest my-website classic
```

啟動網站：

```bash
cd my-website
npx docusaurus start
```

開啟 [`http://localhost:3000`](http://localhost:3000) 跟隨教學。

:::tip

立即透過 **[docusaurus.new](https://docusaurus.new)** 在瀏覽器中測試 Docusaurus！

或線上閱讀 **[5分鐘教學](https://tutorial.docusaurus.io)**。

:::

## 從 v1 遷移 {#migrating-from-v1}

Docusaurus v2+ 是基於現代化工具鏈的全面重構版本。自 [v2 正式發布](https://docusaurus.io/blog/2022/08/01/announcing-docusaurus-2.0) 後，我們強烈建議**使用 v2+ 而非已棄用的 v1 版**。

已有[眾多用戶](/showcase)採用 Docusaurus v2+（[趨勢分析](https://www.npmtrends.com/docusaurus-vs-@docusaurus/core)）。

**適用 Docusaurus v2+ 的情境：**

- :white_check_mark: 需要現代化 Jamstack 文件網站
- :white_check_mark: 需要具客戶端路由的單頁應用程式 (SPA)
- :white_check_mark: 需完整 React 與 MDX 功能
- :white_check_mark: 無需支援 IE11

**適用 [Docusaurus v1](https://v1.docusaurus.io/) 的情境：**

- :x: 不需要單頁應用程式 (SPA)
- :x: 需支援 IE11（注意：IE [已終止支援](https://docs.microsoft.com/en-us/lifecycle/products/internet-explorer-11)）

現有 v1 用戶可參閱我們的[遷移指南](./migration/index.mdx)升級至 v2+。

## 功能特色 {#features}

Docusaurus 的設計高度重視開發者與貢獻者的使用體驗。

- ⚛️ **以 React 與💚打造**：
  - 可使用 React 進行擴展與客製化
  - 透過自訂 React 元件全面掌控網站的瀏覽體驗
- 🔌 **插件化架構**：
  - 透過基礎模板快速建站，再搭配進階功能與插件
  - 可開源插件與社群共享
- ✂️ **開發者體驗**：
  - 立即開始撰寫文件
  - 統一配置入口提升協作者維護性
  - 熱重載與閃電般快速的增量建置
  - 基於路由的程式碼與資料分割
  - 輕鬆部署至 GitHub Pages、Netlify、Vercel 等服務

我們的共同目標——協助使用者快速找到所需資訊並更好理解您的產品。我們分享最佳實踐，助您打造優質文件站點。

- 🎯 **SEO 友善**：
  - 為所有可能路徑靜態生成 HTML 檔案
  - 頁面專屬 SEO 優化，讓使用者直接找到相關問題的官方文件
- 📝 **基於 MDX 技術**：
  - 在 Markdown 中嵌入 JSX 與 React 撰寫互動元件
  - 透過即時編輯器分享程式碼，讓使用者當場愛上您的產品
- 🔍 **全站搜尋**：完整內容皆可搜尋
- 💾 **文件版本控制**：保持文件與專案版本同步
- 🌍 **國際化 (i18n)**：支援多語言站點翻譯

Docusaurus v2+ 天生具備極致效能與全面無障礙設計。

- ⚡️ **閃電速度**。遵循 [PRPL 模式](https://developers.google.com/web/fundamentals/performance/prpl-pattern/)確保內容極速載入
- 🦖 **無障礙設計**。注重可訪問性，讓所有使用者平等獲取內容

## 設計原則 {#design-principles}

- **低學習曲線**。API 簡潔易學，即使需編寫更多程式碼，使用者仍能實現大多數功能。寧缺毋濫的抽象設計，避免使用者被迫繞過不當抽象層。必看演講—[最小化 API 表面積](https://www.youtube.com/watch?v=4anAwXYqLG8)
- **直覺化**。專案目錄結構直觀清晰，新增功能時不會感到困惑，採用使用者熟悉的開發模式
- **分層架構**。內容/主題/樣式各層級關注點分離，模組化抽象設計明確
- **合理預設**。為常見場景預設性能優化配置，同時保留覆寫選項
- **無廠商鎖定**。不強制使用預設插件或 CSS（但強烈建議）。僅核心基礎設施（如 React Loadable 和 React Router）因預設優化不可替換，其他如 Markdown 引擎、CSS 框架等架構選擇完全開放

我們相信，了解工具運作原理能提升開發者使用技巧。因此我們詳述 Docusaurus 架構與各元件設計，助使用者深入掌握工具精髓。

## 與其他工具比較 {#comparison-with-other-tools}

在眾多靜態網站生成器中，Docusaurus 專注於文件站點需求，提供豐富開箱即用功能。

我們深入研究其他主流靜態網站生成器，分享比較見解，助您在眾多選擇中做出決策。

### Gatsby {#gatsby}

[Gatsby](https://www.gatsbyjs.com/) 具備豐富功能，擁有龐大的插件生態系統，能實現 Docusaurus 的所有功能。但這自然伴隨著較高的學習曲線。Gatsby 在多方面表現出色，適合構建各類網站。而 Docusaurus 則專注於將一件事做到極致——成為編寫與發布內容的最佳工具。

GraphQL 在 Gatsby 中佔據核心地位，但構建 Gatsby 網站並非必須使用 GraphQL。多數靜態網站場景中，您並不需要 GraphQL 提供的靈活性。

Docusaurus v2+ 的許多設計靈感源自 Gatsby 的優秀特性，它是一個絕佳的替代方案。

[Docz](https://github.com/pedronauck/docz) 是基於 Gatsby 的主題，用於構建文檔網站。其功能目前少於 Docusaurus。

### Next.js {#nextjs}

[Next.js](https://nextjs.org/) 是另一個熱門的混合式 React 框架。它雖能協助構建文檔網站，但未針對文檔場景進行優化，需投入更多工作量才能實現 Docusaurus 的開箱即用功能。

[Nextra](https://github.com/shuding/nextra) 是基於 Next.js 的靜態網站生成器，具有明確的設計導向。其功能目前少於 Docusaurus。

### VitePress {#vitepress}

[VitePress](https://vitepress.dev/) 與 Docusaurus 有諸多相似之處——兩者都專注於內容型網站，並提供開箱即用的文檔功能。但 VitePress 基於 Vue 構建，而 Docusaurus 基於 React。若您需要 Vue 技術棧方案，VitePress 會是不錯的選擇。

### MkDocs {#mkdocs}

[MkDocs](https://www.mkdocs.org/) 是流行的 Python 靜態網站生成器，其核心價值與 Docusaurus 相似。

如果您不需要單頁應用程式且不計劃使用 React，這將是個好選擇。

[Material for MkDocs](https://squidfunk.github.io/mkdocs-material/) 提供了精美的主題。

### Docsify {#docsify}

[Docsify](https://docsify.js.org/) 能快速創建文檔網站，但它並非靜態網站生成器，且對 SEO 不友好。

### GitBook {#gitbook}

[GitBook](https://www.gitbook.com/) 擁有簡潔設計，曾被眾多開源項目採用。隨著其產品重心轉向商業化而非開源工具，許多功能已不再符合開源項目文檔站點的需求。因此許多項目已遷移至其他方案，您可閱讀 [Redux 遷移至 Docusaurus 的討論](https://github.com/reduxjs/redux/issues/3161)。

目前 GitBook 僅對開源和非營利團隊免費，而 Docusaurus 對所有人免費。

### Jekyll {#jekyll}

[Jekyll](https://github.com/jekyll/jekyll) 是最成熟的靜態網站生成器之一——事實上在 Docusaurus 之前，Facebook 多數開源網站都基於 Jekyll 構建！它的入門極其簡單，我們希望提供與 Jekyll 相似的開發體驗。

相較於通過 `<script />` 標籤添加交互性的靜態 HTML，Docusaurus 網站是 React 應用程式。藉助現代 JavaScript 生態工具鏈，我們期望在文檔站點的效能、資源構建流程優化及易用性方面樹立新標準。
