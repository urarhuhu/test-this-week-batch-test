元件可以透過 Flexbox 演算法來指定其子元素的佈局方式。Flexbox 旨在為不同螢幕尺寸提供一致的佈局。

通常你會結合使用 `flexDirection`、`alignItems` 和 `justifyContent` 來達成正確的佈局。

:::caution
Flexbox 在 React Native 中的運作方式與在 CSS 網頁上相同，但有幾個例外。
預設值不同：`flexDirection` 預設為 `column` 而非 `row`，`alignContent` 預設為 `flex-start` 而非 `stretch`，`flexShrink` 預設為 `0` 而非 `1`，且 `flex` 參數僅支援單一數值。
:::

## Flex 彈性佈局

[`flex`](layout-props#flex) 會定義你的項目如何沿著主軸「填滿」可用空間。空間將根據每個元素的 flex 屬性進行分配。

在以下範例中，紅色、橙色和綠色視圖都是設有 `flex: 1` 的容器視圖的子元素。紅色視圖使用 `flex: 1`，橙色視圖使用 `flex: 2`，綠色視圖使用 `flex: 3`。**1+2+3 = 6**，這意味著紅色視圖將獲得 `1/6` 的空間，橙色視圖獲得 `2/6`，綠色視圖獲得 `3/6`。

```SnackPlayer name=Flex%20Example
import React from 'react';
import {StyleSheet, View} from 'react-native';

const Flex = () => {
  return (
    <View
      style={[
        styles.container,
        {
          // Try setting `flexDirection` to `"row"`.
          flexDirection: 'column',
        },
      ]}>
      <View style={{flex: 1, backgroundColor: 'red'}} />
      <View style={{flex: 2, backgroundColor: 'darkorange'}} />
      <View style={{flex: 3, backgroundColor: 'green'}} />
    </View>
  );
};

const styles = StyleSheet.create({
  container: {
    flex: 1,
    padding: 20,
  },
});

export default Flex;
```

## Flex Direction 主軸方向

[`flexDirection`](layout-props#flexdirection) 控制節點子元素的排列方向，這也被稱為主軸。交叉軸則是與主軸垂直的軸，或是換行排列的方向。

- `column` (**預設值**) 從上到下排列子元素。如果啟用換行，則下一行將從容器頂部第一個項目的右側開始。
- `row` 從左到右排列子元素。如果啟用換行，則下一行將從容器左側第一個項目的下方開始。
- `column-reverse` 從下到上排列子元素。如果啟用換行，則下一行將從容器底部第一個項目的右側開始。
- `row-reverse` 從右到左排列子元素。如果啟用換行，則下一行將從容器右側第一個項目的下方開始。

你可以在此處了解更多：[Flex Direction](https://www.yogalayout.dev/docs/styling/flex-direction)。

## Layout Direction 佈局方向

佈局 [`direction`](layout-props#direction) 指定了子元素和文字在層次結構中的排列方向。佈局方向也會影響 `start` 和 `end` 所指向的邊緣。預設情況下，React Native 使用 LTR（從左到右）的佈局方向。在此模式下，`start` 指向左側，`end` 指向右側。

- `LTR` (**預設值**) 文字和子元素從左到右排列。應用於元素起始位置的 margin 和 padding 會作用於左側。
- `RTL` 文字和子元素從右到左排列。應用於元素起始位置的 margin 和 padding 會作用於右側。

## Justify Content 主軸對齊

[`justifyContent`](layout-props#justifycontent) 描述了如何在容器的主軸上對齊子元素。例如，你可以使用此屬性在 `flexDirection` 設為 `row` 的容器中水平居中子元素，或在 `flexDirection` 設為 `column` 的容器中垂直居中子元素。

- `flex-start`（**預設值**）將容器的子元素對齊至主軸的起始端。

- `flex-end` 將容器的子元素對齊至主軸的末端。

- `center` 將容器的子元素在主軸上居中對齊。

- `space-between` 在主軸上均勻分布子元素，剩餘空間分配於子元素之間。

- `space-around` 在主軸上均勻分布子元素，剩餘空間環繞子元素分配。與 `space-between` 相比，`space-around` 會在首個子元素前和最後子元素後也分配空間。

- `space-evenly` 在主軸上完全均勻分布子元素。每對相鄰子元素之間、主軸起始端與首個子元素之間、主軸末端與最後子元素之間的間距均完全相同。

你可以在此[了解更多](https://www.yogalayout.dev/docs/styling/justify-content)。

## 對齊項目 (Align Items)

[`alignItems`](layout-props#alignitems) 描述如何沿容器的交叉軸對齊子元素。其功能與 `justifyContent` 類似，但作用於交叉軸而非主軸。

- `stretch`（**預設值**）拉伸容器的子元素以匹配容器交叉軸的 `height`。

- `flex-start` 將容器的子元素對齊至交叉軸的起始端。

- `flex-end` 將容器的子元素對齊至交叉軸的末端。

- `center` 將容器的子元素在交叉軸上居中對齊。

- `baseline` 沿共同基線對齊容器的子元素。可將個別子元素設為父元素的基準線。

:::info
要使 `stretch` 生效，子元素在次要軸上不能有固定尺寸。以下範例中，除非移除子元素的 `width: 50` 設定，否則 `alignItems: stretch` 不會產生效果。
:::

你可以在此[了解更多](https://www.yogalayout.dev/docs/styling/align-items-self)。

## 對齊內容 (Align Content)

[alignContent](layout-props#aligncontent) 定義多行內容在交叉軸上的分布方式，僅在使用 `flexWrap` 換行時生效。

- `flex-start`（**預設值**）將換行內容對齊至容器交叉軸的起始端。

- `flex-end` 將換行內容對齊至容器交叉軸的末端。

- `stretch`（_網頁版 Yoga 的預設值_）拉伸換行內容以匹配容器交叉軸的高度。

- `center` 將換行內容在交叉軸上居中對齊。

- `space-between` 在交叉軸上均勻分布換行內容，剩餘空間分配於行與行之間。

- `space-around` 在交叉軸上均勻分布換行內容，剩餘空間環繞各行分配。容器兩端的空間為行間距的一半。

- `space-evenly` 在交叉軸上完全均勻分布換行內容，所有間距（包括容器兩端）大小相同。

你可以在此[了解更多](https://www.yogalayout.dev/docs/styling/align-content)。

## Flex Basis、Grow 與 Shrink

- [`flexBasis`](layout-props#flexbasis) 是一種與軸向無關的方式，用於設定項目在主軸上的默認尺寸。設置子元素的 `flexBasis` 類似於：當父容器為 `flexDirection: row` 時設定該子元素的 `width`，或當父容器為 `flexDirection: column` 時設定其 `height`。項目的 `flexBasis` 是該項目的默認尺寸，即在進行任何 `flexGrow` 和 `flexShrink` 計算之前的尺寸。

- [`flexGrow`](layout-props#flexgrow) 描述容器內的主軸剩餘空間應如何分配給其子元素。在佈局子元素後，容器會根據子元素指定的 flex grow 值分配剩餘空間。

  `flexGrow` 接受任何大於等於 0 的浮點數，默認值為 0。容器會按照子元素的 `flexGrow` 值加權分配剩餘空間。

- [`flexShrink`](layout-props#flexshrink) 描述當子元素的總尺寸在主軸上超出容器尺寸時，應如何縮小子元素。`flexShrink` 與 `flexGrow` 非常相似，可以將溢出的尺寸視為負的剩餘空間來理解。這兩個屬性協同工作，使子元素能根據需要伸縮。

  `flexShrink` 接受任何大於等於 0 的浮點數，默認值為 0（在網頁上默認為 1）。容器會按照子元素的 `flexShrink` 值加權縮小子元素。

了解更多[請參閱此處](https://www.yogalayout.dev/docs/styling/flex-basis-grow-shrink)。

## 寬度與高度

`width` 屬性指定元素內容區域的寬度，`height` 屬性則指定元素內容區域的高度。

`width` 和 `height` 可接受以下值：

- `auto`（**默認值**）React Native 會根據元素的內容（無論是其他子元素、文字還是圖片）計算其寬度/高度。
- `pixels` 以絕對像素單位定義寬度/高度。根據組件上設置的其他樣式，這可能並非節點的最終尺寸。
- `percentage` 以父元素寬度或高度的百分比分別定義寬度或高度。

## 定位

元素的 `position` 類型定義了其相對於自身、父元素或[包含塊](./flexbox.md#the-containing-block)的定位方式。

- `relative`（**預設值**）預設情況下，元素採用相對定位。這表示元素會依照正常的版面流進行定位，然後根據 `top`、`right`、`bottom` 和 `left` 的值相對於該位置進行偏移。此偏移不會影響任何兄弟元素或父元素的位置。

- `absolute` 當元素採用絕對定位時，它不會參與正常的版面流，而是獨立於其兄弟元素進行布局。其位置由 `top`、`right`、`bottom` 和 `left` 的值決定，這些值會將元素定位相對於其包含區塊（containing block）。

- `static` 當元素採用靜態定位時，它會依照正常的版面流進行定位，並忽略 `top`、`right`、`bottom` 和 `left` 的值。此 `position` 設定也會導致該元素不會為其絕對定位的子元素形成包含區塊，除非存在其他優先的樣式屬性（例如 `transform`）。這允許 `absolute` 元素可以定位到非其父元素的區塊。請注意，**`static` 僅在新架構（New Architecture）中可用**。

## 包含區塊（Containing Block）

元素的包含區塊是指控制其位置和大小的祖先元素。  
React Native 中包含區塊的運作方式與 [網頁上的運作方式](https://developer.mozilla.org/en-US/docs/Web/CSS/Containing_block) 非常相似，但由於缺少某些網頁功能而有所簡化。

絕對定位元素的 `top`、`right`、`bottom` 和 `left` 值將相對於其包含區塊計算。

套用於絕對定位元素的百分比長度（例如 `width: '50%'` 或 `padding: '10%'`）將根據其包含區塊的大小計算。例如，如果包含區塊的寬度為 100 點，則絕對定位元素上的 `width: 50%` 會使其寬度為 50 點。

以下列表可幫助您判斷任何元素的包含區塊：

- 如果該元素的 `position` 類型為 `relative` 或 `static`，則其包含區塊為其父元素。
- 如果該元素的 `position` 類型為 `absolute`，則其包含區塊為滿足以下條件之一的最近祖先元素：
  - 其 `position` 類型不為 `static`
  - 其具有 `transform` 屬性
