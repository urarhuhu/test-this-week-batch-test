# MDX 與 React

```mdx-code-block
import BrowserWindow from '@site/src/components/BrowserWindow';
import Tabs from '@theme/Tabs';
import TabItem from '@theme/TabItem';
import styles from './markdown-features-react.module.css';
```

Docusaurus 內建支援 [MDX](https://mdxjs.com/)，允許您在 Markdown 檔案中編寫 JSX 並將其渲染為 React 元件。

請參閱 [MDX 文件](https://mdxjs.com/) 了解您能用 MDX 實現哪些進階功能。

:::tip[除錯 MDX]

MDX 格式相當嚴格，您可能會遇到編譯錯誤。

使用 **[MDX 遊樂場](https://mdxjs.com/playground/)** 進行除錯，確保您的語法是有效的。

:::

:::info

最流行的格式化工具 Prettier [僅支援舊版 MDX v1](https://github.com/prettier/prettier/issues/12209)。如果您遇到非預期的格式化結果，可以在問題區域前添加 `{/* prettier-ignore */}`，或將 `*.mdx` 加入您的 `.prettierignore` 中，直到 Prettier 完整支援 MDX v3。[MDX 的主要作者之一推薦使用 `remark-cli` 搭配 `remark-mdx`](https://github.com/orgs/mdx-js/discussions/2067)。

:::

### 匯出元件 {#exporting-components}

要在 MDX 檔案中定義任何自訂元件，您必須將其匯出：只有以 `export` 開頭的段落才會被解析為元件而非普通文字。

```jsx
export const Highlight = ({children, color}) => (
  <span
    style={{
      backgroundColor: color,
      borderRadius: '2px',
      color: '#fff',
      padding: '0.2rem',
    }}>
    {children}
  </span>
);

<Highlight color="#25c2a0">Docusaurus green</Highlight> and <Highlight color="#1877F2">Facebook blue</Highlight> are my favorite colors.

I can write **Markdown** alongside my _JSX_!
```

請注意它如何同時渲染您的 React 元件的標記和 Markdown 語法：

```mdx-code-block
export const Highlight = ({children, color}) => (
  <span
    style={{
      backgroundColor: color,
      borderRadius: '2px',
      color: '#fff',
      padding: '0.2rem',
    }}>
    {children}
  </span>
);

<BrowserWindow minHeight={240}>

<><Highlight color="#25c2a0">Docusaurus green</Highlight>
{` `}and <Highlight color="#1877F2">Facebook blue</Highlight> are my favorite colors.</>

I can write **Markdown** alongside my _JSX_!

</BrowserWindow>
```

:::warning[MDX 即 JSX]

由於所有文件都是使用 MDX 解析的，任何看似 HTML 的內容實際上是 JSX。因此，如果您需要內聯樣式元件，請遵循 JSX 風格並提供樣式物件。

{/* prettier-ignore */}
```jsx
/* 不要這樣做： */
<span style="background-color: red">Foo</span>
/* 請這樣做： */
<span style={{backgroundColor: 'red'}}>Foo</span>
```

:::

### 匯入元件 {#importing-components}

您也可以匯入在其他檔案中定義的自訂元件，或透過 npm 安裝的第三方元件。

```md
<!-- Docusaurus theme component -->
import TOCInline from '@theme/TOCInline';
<!-- External component -->
import Button from '@mui/material/Button';
<!-- Custom component -->
import BrowserWindow from '@site/src/components/BrowserWindow';
```

:::tip

`@site` 別名指向您網站的目錄，通常是 `docusaurus.config.js` 檔案所在的位置。使用別名而非相對路徑（如 `'../../src/components/BrowserWindow'`）可以避免在移動檔案或進行[文件版本控制](../docs/versioning.mdx)和[翻譯](../../i18n/i18n-tutorial.mdx)時更新匯入路徑。

:::

雖然在 Markdown 中宣告元件對於簡單情況非常方便，但由於編輯器支援有限、解析錯誤的風險以及可重用性低，維護起來會變得困難。當您的元件涉及複雜的 JS 邏輯時，請使用獨立的 `.js` 檔案：

```jsx title="src/components/Highlight.js"
import React from 'react';

export default function Highlight({children, color}) {
  return (
    <span
      style={{
        backgroundColor: color,
        borderRadius: '2px',
        color: '#fff',
        padding: '0.2rem',
      }}>
      {children}
    </span>
  );
}
```

```md title="markdown-file.mdx"
import Highlight from '@site/src/components/Highlight';

<Highlight color="#25c2a0">Docusaurus green</Highlight>
```

:::tip

如果您在多個檔案中使用相同的元件，不需要在每個地方都匯入它——考慮將其加入全域範圍。[請參閱下方](#mdx-component-scope)

:::

### MDX 元件範圍 {#mdx-component-scope}

除了[匯入元件](#importing-components)和[匯出元件](#exporting-components)之外，第三種在 MDX 中使用元件的方法是**將其註冊到全域範圍**，這將使其自動在所有 MDX 檔案中可用，無需任何匯入語句。

例如，給定以下 MDX 檔案：

```md
- a
- list!

And some <Highlight>custom markup</Highlight>...
```

它將被編譯為包含 `ul`、`li`、`p` 和 `Highlight` 元素的 React 元件。`Highlight` 不是原生的 HTML 元素：您需要為其提供自己的 React 元件實現。

在 Docusaurus 中，MDX 元件作用域由 `@theme/MDXComponents` 檔案提供。嚴格來說它並非 React 元件（與大多數 `@theme/` 別名下的匯出不同），而是一個從標籤名稱（如 `Highlight`）映射到其實作 React 元件的紀錄。

若您[反轉](../../swizzling.mdx)此元件，將會看到所有已實作的標籤，並可透過反轉相應子元件（如用於渲染[Markdown 程式碼區塊](./markdown-features-code-blocks.mdx)的 `@theme/MDXComponents/Code`）進一步客製化實作。

若要註冊額外標籤名稱（如上述 `<Highlight>` 標籤），建議[包裝 `@theme/MDXComponents`](../../swizzling.mdx#wrapping)，如此便無需維護所有現有映射。由於反轉 CLI 尚不支援包裝非元件檔案，您需手動建立包裝器：

```js title="src/theme/MDXComponents.js"
import React from 'react';
// Import the original mapper
import MDXComponents from '@theme-original/MDXComponents';
// highlight-next-line
import Highlight from '@site/src/components/Highlight';

export default {
  // Re-use the default mapping
  ...MDXComponents,
  // Map the "<Highlight>" tag to our Highlight component
  // `Highlight` will receive all props that were passed to `<Highlight>` in MDX
  // highlight-next-line
  Highlight,
};
```

現在您可在所有頁面直接使用 `<Highlight>`，無需撰寫匯入語句：

```md
I can conveniently use <Highlight color="#25c2a0">Docusaurus green</Highlight> everywhere!
```

```mdx-code-block
<BrowserWindow>

I can conveniently use <Highlight color="#25c2a0">Docusaurus green</Highlight> everywhere!

</BrowserWindow>
```

:::warning

我們刻意採用**大寫**標籤名稱如 `Highlight`。

從 MDX v3+ 開始（Docusaurus v3+），小寫標籤名稱一律會被渲染為原生 HTML 元素，且不會使用您提供的任何元件映射。

:::

:::warning

此功能由 [`MDXProvider`](https://mdxjs.com/docs/using-mdx/#mdx-provider) 驅動。若您在 React 頁面匯入 Markdown，需透過 `MDXContent` 主題元件自行提供此供應器：

```jsx title="src/pages/index.js"
import React from 'react';
import FeatureDisplay from './_featureDisplay.mdx';
// highlight-next-line
import MDXContent from '@theme/MDXContent';

export default function LandingPage() {
  return (
    <div>
      {/* highlight-start */}
      <MDXContent>
        <FeatureDisplay />
      </MDXContent>
      {/* highlight-end */}
    </div>
  );
}
```

若未用 `MDXContent` 包裹匯入的 MDX，全域作用域將無法生效。

:::

### Markdown 與 JSX 互通性 {#markdown-and-jsx-interoperability}

Docusaurus v3 採用 [MDX v3](https://mdxjs.com/blog/v3/)。

[MDX 語法](https://mdxjs.com/docs/what-is-mdx/#mdx-syntax)大體相容於 [CommonMark](https://commonmark.org/)，但更為嚴格，因為 `.mdx` 檔案可使用 JSX 並會被編譯為真實 React 元件（參見[互動式遊樂場](https://mdxjs.com/playground/)）。

部分 CommonMark 有效功能在 MDX 中無法運作（[詳情](https://mdxjs.com/docs/what-is-mdx/#markdown)），特別是：

- 縮排式程式碼區塊：請改用三重反引號
- 自動連結（`<http://localhost:3000>`）：請改用標準連結語法（`[http://localhost:3000](http://localhost:3000)`）
- HTML 語法（`<p style="color: red;">`）：請改用 JSX（`<p style={{color: 'red'}}>`）
- 未跳脫的 `{` 和 `<`：請用 `\` 跳脫（`\{` 和 `\<`）

:::danger[實驗性 CommonMark 支援]

Docusaurus v3 可透過以下選項選擇啟用較寬鬆的標準 [CommonMark](https://commonmark.org/) 支援：

- 在 front matter 設定 `mdx.format: md`
- 使用 `.md` 副檔名並搭配 `siteConfig.markdown.format: "detect"` 配置

此功能屬**實驗性質**，目前存在若干[限制](https://github.com/facebook/docusaurus/issues/9092)。

:::

## 匯入程式碼片段 {#importing-code-snippets}

您不僅可以導入包含組件定義的文件，還能透過 [Webpack raw-loader](https://webpack.js.org/loaders/raw-loader/) 將任何代碼文件作為原始文本導入，並插入到代碼區塊中。要使用 `raw-loader`，首先需在專案中安裝它：

```bash npm2yarn
npm install --save raw-loader
```

現在您可以直接導入其他文件中的代碼片段：

```jsx title="myMarkdownFile.mdx"
import CodeBlock from '@theme/CodeBlock';
import MyComponentSource from '!!raw-loader!./myComponent';

<CodeBlock language="jsx">{MyComponentSource}</CodeBlock>
```

```mdx-code-block
import CodeBlock from '@theme/CodeBlock';
import MyComponentSource from '!!raw-loader!@site/src/pages/examples/_myComponent';

<BrowserWindow>

<CodeBlock language="jsx">{MyComponentSource}</CodeBlock>

</BrowserWindow>
```

詳見 [JSX 中使用代碼區塊](./markdown-features-code-blocks.mdx#usage-in-jsx) 以了解 `<CodeBlock>` 組件的更多細節。

:::note

此處必須使用 `<CodeBlock>` 而非 Markdown 的三重反引號 ` ``` `，因為後者會將其內容原樣輸出，而您需要在此處插入導入的文本。

:::

:::warning

此功能為實驗性質，未來可能會有破壞性 API 變更。

:::

## 導入 Markdown 文件 {#importing-markdown}

您可以將 Markdown 文件作為組件導入其他 Markdown 文件或 React 頁面。每個 MDX 文件會將其頁面內容作為 React 組件默認導出。在 `import` 語句中，您可以用任何符合 React 命名規則的大寫名稱來默認導入此組件。

按照約定，使用 **`_` 文件名前綴** 將不會創建任何文檔頁面，表示該 Markdown 文件是供其他文件導入的 **「部分內容」**。

```md title="_markdown-partial-example.mdx"
<span>Hello {props.name}</span>

This is text some content from `_markdown-partial-example.mdx`.
```

```jsx title="someOtherDoc.mdx"
import PartialExample from './_markdown-partial-example.mdx';

<PartialExample name="Sebastien" />
```

```mdx-code-block
import PartialExample from './_markdown-partial-example.mdx';

<BrowserWindow>
  <PartialExample name="Sebastien" />
</BrowserWindow>
```

透過這種方式，您可以在多個頁面間複用內容，避免材料重複。

## 可用導出變量 {#available-exports}

在 MDX 頁面中，以下變量可作為全局變量使用：

- `frontMatter`: 前端參數，為字符串鍵值對的記錄；
- `toc`: 目錄結構，以標題樹的形式呈現。另請參見 [內聯目錄](./markdown-features-toc.mdx#inline-table-of-contents) 以獲得更具體的使用案例。
- `contentTitle`: Markdown 標題，即 Markdown 文本中的第一個 `h1` 標題。若不存在則為 `undefined`（例如在 front matter 中指定了標題）。

```jsx
import TOCInline from '@theme/TOCInline';
import CodeBlock from '@theme/CodeBlock';

The table of contents for this page, serialized:

<CodeBlock className="language-json">{JSON.stringify(toc, null, 2)}</CodeBlock>

The front matter of this page:

<ul>
  {Object.entries(frontMatter).map(([key, value]) => <li key={key}><b>{key}</b>: {value}</li>)}
</ul>

<p>The title of this page is: <b>{contentTitle}</b></p>
```

```mdx-code-block
import TOCInline from '@theme/TOCInline';

<BrowserWindow>

The table of contents for this page, serialized:

<CodeBlock className="language-json">{JSON.stringify(toc, null, 2)}</CodeBlock>

The front matter of this page:

<ul>
  {Object.entries(frontMatter).map(([key, value]) => <li key={key}><b>{key}</b>: {value}</li>)}
</ul>

<p>The title of this page is: <b>{contentTitle}</b></p>

</BrowserWindow>
```
