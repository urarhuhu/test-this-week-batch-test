# 建立文件

建立一個 Markdown 檔案 `greeting.md`，並將其放置在 `docs` 目錄下。

```bash
website # root directory of your site
├── docs
│   └── greeting.md
├── src
│   └── pages
├── docusaurus.config.js
├── ...
```

```md
---
description: Create a doc page with rich content.
---

# Hello from Docusaurus

Are you ready to create the documentation site for your open source project?

## Headers

will show up on the table of contents on the upper right

So that your users will know what this page is all about without scrolling down or even without reading too much.

## Only h2 and h3 will be in the TOC by default.

You can configure the TOC heading levels either per-document or in the theme configuration.

The headers are well-spaced so that the hierarchy is clear.

- lists will help you
- present the key points
- that you want your users to remember
  - and you may nest them
    - multiple times
```

:::note

預設情況下，`docs` 目錄中所有以下劃線 (`_`) 為前綴的檔案都會被視為「部分」頁面並被忽略。

詳見[導入部分頁面](../markdown-features/markdown-features-react.mdx#importing-markdown)。

:::

## 文件 Front Matter {#doc-front-matter}

[Front Matter](../markdown-features/markdown-features-intro.mdx#front-matter) 用於為文件頁面提供額外的元數據。Front Matter 是可選的——Docusaurus 能夠在沒有 Front Matter 的情況下推斷所有必要的元數據。例如，下文介紹的[文件標籤](#doc-tags)功能需要使用 Front Matter。所有可能的字段詳見 [API 文檔](../../api/plugins/plugin-content-docs.mdx#markdown-front-matter)。

## 文件標籤 {#doc-tags}

標籤在 Front Matter 中聲明，為[文件側邊欄](./sidebar/index.mdx)之外提供了另一種分類維度。

可以定義內聯標籤，或引用在 [`tags 文件`](../../api/plugins/plugin-content-docs.mdx#tags-file)（可選，通常為 `docs/tags.yml`）中預先定義的標籤。

以下範例中：

- `docusaurus` 引用在 `docs/tags.yml` 中預定義的標籤鍵
- `Releases` 是一個內聯標籤，因為它不存在於 `docs/tags.yml` 中

```md title="docs/my-doc.md"
---
tags:
  - Releases
  - docusaurus
---

# Title

Content
```

```yml title="docs/tags.yml"
docusaurus:
  label: 'Docusaurus'
  permalink: '/docusaurus'
  description: 'Docs related to the Docusaurus framework'
```

:::tip

標籤也可以使用 `tags: [Demo, Getting started]` 的形式聲明。

詳見所有可能的 [Yaml 陣列語法](https://www.w3schools.io/file/yaml-arrays/)。

:::

## 組織檔案結構 {#organizing-folder-structure}

Markdown 檔案在 `docs` 資料夾下的排列方式可能會對 Docusaurus 內容生成產生多種影響。但大多數情況下可以與檔案結構解耦。

### 文件 ID {#document-id}

每個文件都有一個唯一的 `id`。預設情況下，文件 `id` 是相對於根 docs 目錄的文件名（不包含副檔名）。

例如，`greeting.md` 的 ID 是 `greeting`，而 `guide/hello.md` 的 ID 是 `guide/hello`。

```bash
website # Root directory of your site
└── docs
   ├── greeting.md
   └── guide
      └── hello.md
```

但用戶可以在 Front Matter 中定義 `id` 的**最後部分**。例如，如果 `guide/hello.md` 的內容如下定義，其最終 `id` 為 `guide/part1`。

```md
---
id: part1
---

Lorem ipsum
```

ID 用於在手寫側邊欄或使用與文件相關的佈局組件或鉤子時引用文件。

### 文件 URL {#doc-urls}

預設情況下，文件的 URL 位置是其相對於 `docs` 資料夾的檔案路徑，但有幾個例外。如果檔案名為以下之一，則檔案名不會包含在 URL 中：

- 命名為 `index`（不區分大小寫）：`docs/Guides/index.md`
- 命名為 `README`（不區分大小寫）：`docs/Guides/README.mdx`
- 與父資料夾同名：`docs/Guides/Guides.md`

在所有情況下，預設的 slug 僅為 `/Guides`，不包含 `/index`、`/README` 或重複的 `/Guides` 部分。

:::note

此慣例與[類別索引慣例](./sidebar/autogenerated.mdx#category-index-convention)完全相同。但需注意，`isCategoryIndex`配置項_不會_影響文件URL的生成。

:::

使用`slug`前置元數據可修改文件的URL路徑。

例如，假設您的站點結構如下：

```bash
website # Root directory of your site
└── docs
    └── guide
        └── hello.md
```

默認情況下`hello.md`的訪問路徑將是`/docs/guide/hello`。您可以通過以下方式將其URL修改為`/docs/bonjour`：

```md
---
slug: /bonjour
---

Lorem ipsum
```

`slug`值會附加到文檔插件的`routeBasePath`基礎路徑後（默認為`/docs`）。如需移除URL中的`/docs`前綴，請參閱[純文檔模式](docs-introduction.mdx#docs-only-mode)的配置方法。

:::note

支持以下兩種slug格式：

- 絕對路徑：`slug: /mySlug`, `slug: /`...
- 相對路徑：`slug: mySlug`, `slug: ./../mySlug`...

:::

若需將文檔設置為站點根路徑（如`https://docusaurus.io/docs/`），可使用slug前置元數據：

```md
---
id: my-home-doc
slug: /
---

Lorem ipsum
```

### 側邊欄 {#sidebars}

使用[自動生成側邊欄](./sidebar/autogenerated.mdx)時，文件結構將決定側邊欄的層級結構。

我們推薦的文件組織原則：使文件系統結構與側邊欄結構保持一致（這樣就無需手動編寫`sidebars.js`文件），並通過`slug`前置元數據來定制每個文檔的URL路徑。
