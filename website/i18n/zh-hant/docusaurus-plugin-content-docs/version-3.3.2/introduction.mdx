# 介紹

⚡️ Docusaurus 能協助您快速打造出**美觀的技術文件網站**。

💸 從零建構技術堆疊成本高昂。請**專注於內容產出**，只需撰寫 Markdown 檔案即可。

💥 需要更多功能？可使用**進階特性**如版本控制、國際化、站內搜尋與主題客製化。

💅 查看**[精選 Docusaurus 網站](/showcase?tags=favorite)**獲取靈感，並閱讀**[使用者見證](https://x.com/sebastienlorber/timelines/1392048416872706049)**。

🧐 Docusaurus 是**靜態網站生成器**。它建構出具有快速客戶端導航的**單頁應用程式**，充分發揮 **React** 的互動性優勢。除預設的**技術文件功能**外，亦可用於建立**任何類型網站**（個人網站、產品頁、部落格、行銷登陸頁等）。

![](/img/slash-introducing.svg)

## 快速上手 ⏱️ {#fast-track}

透過實作在 **5 分鐘** 內理解 Docusaurus！

建立新 Docusaurus 網站並遵循內建的**極簡教學**。

安裝 [Node.js](https://nodejs.org/en/download/) 後建立新網站：

```bash
npx create-docusaurus@latest my-website classic
```

啟動網站：

```bash
cd my-website
npx docusaurus start
```

開啟 [`http://localhost:3000`](http://localhost:3000) 並跟隨教學指引。

:::tip

立即透過 **[docusaurus.new](https://docusaurus.new)** 在瀏覽器中測試 Docusaurus！

或線上閱讀 **[5 分鐘教學](https://tutorial.docusaurus.io)**。

:::

## 從 v1 遷移 {#migrating-from-v1}

Docusaurus v2+ 是基於現代化工具鏈的完全重寫版本。自 [v2 正式發布](https://docusaurus.io/blog/2022/08/01/announcing-docusaurus-2.0) 後，我們強烈建議**使用 v2+ 而非已棄用的 v1 版本**。

已有[眾多使用者](/showcase)採用 Docusaurus v2+（參見[趨勢分析](https://www.npmtrends.com/docusaurus-vs-@docusaurus/core)）。

**適用 Docusaurus v2+ 的情境：**

- :white_check_mark: 需要現代化 Jamstack 技術文件網站
- :white_check_mark: 需具備客戶端路由的單頁應用程式 (SPA)
- :white_check_mark: 需完整 React 與 MDX 功能支援
- :white_check_mark: 無需支援 IE11 瀏覽器

**適用 [Docusaurus v1](https://v1.docusaurus.io/) 的情境：**

- :x: 不需要單頁應用程式 (SPA)
- :x: 需支援 IE11（請注意 IE [已停止官方支援](https://docs.microsoft.com/en-us/lifecycle/products/internet-explorer-11)）

現有 v1 使用者可參閱我們的[遷移指南](./migration/index.mdx)升級至 v2+。

## 功能特色 {#features}

Docusaurus 的設計高度重視開發者與貢獻者的使用體驗。

- ⚛️ **以 React 與 💚 打造**：
  - 透過 React 進行擴展與客製化
  - 提供自訂 React 元件來完全掌控網站的瀏覽體驗
- **可插拔式架構**：
  - 使用基礎模板快速建立網站，再透過進階功能與插件強化
  - 將插件開源以與社群共享
- ✂️ **開發者體驗**：
  - 立即開始撰寫文件
  - 通用配置入口提升協作者維護性
  - 熱重載與閃電般快速的增量建置
  - 基於路由的程式碼與資料分割
  - 輕鬆部署至 GitHub Pages、Netlify、Vercel 等服務

我們的共同目標——幫助使用者快速找到所需內容並更好理解您的產品。我們分享最佳實踐，助您打造優質文件站點。

- 🎯 **SEO 優化**：
  - 為所有可能路徑靜態生成 HTML 檔案
  - 頁面專屬 SEO 讓使用者直接找到相關問題的官方文件
- 📝 **基於 MDX**：
  - 在 Markdown 中嵌入 JSX 和 React 撰寫互動元件
  - 透過即時編輯器分享程式碼，讓使用者當場愛上您的產品
- 🔍 **全站搜尋**：完整內容皆可搜尋
- 💾 **文件版本控制**：保持文件與專案版本同步
- 🌍 **國際化 (i18n)**：支援多語言站點翻譯

Docusaurus v2+ 天生具備極致效能，並貼心照顧所有使用者的無障礙需求。

- ⚡️ **閃電速度**。遵循 [PRPL 模式](https://developers.google.com/web/fundamentals/performance/prpl-pattern/)，確保內容極速載入
- 🦖 **無障礙設計**。注重可訪問性，讓所有使用者平等獲取內容

## 設計原則 {#design-principles}

- **低學習曲線**。API 簡潔易學，即使需更多程式碼與時間，使用者仍能實現大多數功能。寧缺毋濫的抽象設計，避免使用者被迫繞過不當抽象層。必看演講—[最小化 API 表面積](https://www.youtube.com/watch?v=4anAwXYqLG8)。
- **直覺化**。查看專案目錄或新增功能時不會感到壓力，採用熟悉的方式直觀擴展。
- **分層架構**。各層級（內容/主題/樣式）關注點分離，抽象完善且模組化。
- **合理預設**。為常見場景預設性能優化配置，同時保留覆寫選項。
- **無廠商鎖定**。不強制使用預設插件或 CSS（但強烈建議）。核心基礎如 React Loadable 和 React Router 因預設優化不可替換，但高階架構如 Markdown 引擎、CSS 框架等完全由使用者選擇。

我們相信，理解工具運作原理能提升使用效能。因此我們詳述架構設計，助使用者深入掌握 Docusaurus。

## 與其他工具比較 {#comparison-with-other-tools}

在所有靜態網站生成器中，Docusaurus 專注於文件站點，提供大量開箱即用功能。

我們研究其他主流生成器後，分享比較見解，助您在多樣化選擇中做出決策。

### Gatsby {#gatsby}

[Gatsby](https://www.gatsbyjs.com/) 具備豐富功能與龐大插件生態系統，能實現 Docusaurus 的所有功能，但代價是較高的學習曲線。Gatsby 擅長建置各類網站，而 Docusaurus 則專注於將單一任務做到極致——成為編寫與發布內容的最佳工具。

GraphQL 是 Gatsby 的核心技術，但建置靜態網站時通常不需要其提供的靈活性。

Docusaurus v2+ 的諸多設計靈感源自 Gatsby 的優點，是極佳的替代方案。

[Docz](https://github.com/pedronauck/docz) 是基於 Gatsby 的主題套件，目前功能完整度不及 Docusaurus。

### Next.js {#nextjs}

[Next.js](https://nextjs.org/) 是熱門的混合式 React 框架，雖能建置文件網站，但缺乏針對文檔場景的預設設計，需投入更多工作量才能實現 Docusaurus 的開箱即用功能。

[Nextra](https://github.com/shuding/nextra) 是基於 Next.js 的靜態網站生成器，目前功能完整度低於 Docusaurus。

### VitePress {#vitepress}

[VitePress](https://vitepress.dev/) 與 Docusaurus 高度相似——皆專注於內容型網站並提供文檔專屬功能，差異在於 VitePress 基於 Vue 而 Docusaurus 基於 React。若需 Vue 技術棧方案，VitePress 是理想選擇。

### MkDocs {#mkdocs}

[MkDocs](https://www.mkdocs.org/) 是熱門的 Python 靜態網站生成器，核心價值主張與 Docusaurus 相近。

若不需要單頁應用且不計劃使用 React，此為理想選項。

[Material for MkDocs](https://squidfunk.github.io/mkdocs-material/) 提供精美的主題樣式。

### Docsify {#docsify}

[Docsify](https://docsify.js.org/) 簡化了文件網站創建流程，但非靜態網站生成器且不利於 SEO。

### GitBook {#gitbook}

[GitBook](https://www.gitbook.com/) 擁有簡潔設計並被眾多開源專案採用。隨著其轉型為商業產品，許多功能已不符合開源文檔需求，促使如 Redux 等專案遷移至 Docusaurus（詳見[此討論](https://github.com/reduxjs/redux/issues/3161)）。

現階段 GitBook 僅對開源與非營利團隊免費，而 Docusaurus 完全開放給所有使用者。

### Jekyll {#jekyll}

[Jekyll](https://github.com/jekyll/jekyll) 是最成熟的靜態網站生成器之一——事實上在 Docusaurus 問世前，多數 Facebook 開源專案網站皆基於 Jekyll 建置！其入門門檻極低，我們希望延續這種開發體驗。

相較於 Jekyll 的靜態 HTML 與 `<script />` 標籤互動模式，Docusaurus 網站是完整的 React 應用程式。透過現代 JavaScript 工具鏈，我們期望重新定義文檔網站的效能標準、資源建置流程與優化機制。
