# 配置

:::info

查閱 [**`docusaurus.config.js` API 參考文檔**](api/docusaurus.config.js.mdx) 以獲取完整的選項列表。

:::

Docusaurus 對配置有獨特的處理方式。我們建議您將網站相關信息集中在一處管理。我們會保護此文件的字段，並確保整個網站都能方便地存取這個數據對象。

維護良好的 `docusaurus.config.js` 能幫助您、協作者和開源貢獻者專注於文檔撰寫，同時仍能自訂網站功能。

## 宣告 `docusaurus.config.js` 的語法 {#syntax-to-declare-docusaurus-config}

`docusaurus.config.js` 文件在 Node.js 環境中執行，應導出以下其中一項：

- 一個**配置對象**
- 一個創建配置對象的**函數**

:::info

`docusaurus.config.js` 文件支援：

- [**ES 模組**](https://flaviocopes.com/es-modules/)
- [**CommonJS**](https://flaviocopes.com/commonjs/)
- [**TypeScript**](./typescript-support.mdx#typing-config)

限制條件：

- **必填項：** 使用 `export default /* 您的配置 */`（或 `module.exports`）導出 Docusaurus 配置
- **可選項：** 使用 `import Lib from 'lib'`（或 `require('lib')`）導入 Node.js 套件

:::

Docusaurus 允許我們以多種**等效方式**宣告配置，以下所有配置範例都會產生完全相同的結果：

```js title="docusaurus.config.js"
export default {
  title: 'Docusaurus',
  url: 'https://docusaurus.io',
  // your site config ...
};
```

```js title="docusaurus.config.js"
module.exports = {
  title: 'Docusaurus',
  url: 'https://docusaurus.io',
  // your site config ...
};
```

```ts title="docusaurus.config.ts"
import type {Config} from '@docusaurus/types';

export default {
  title: 'Docusaurus',
  url: 'https://docusaurus.io',
  // your site config ...
} satisfies Config;
```

```js title="docusaurus.config.js"
const config = {
  title: 'Docusaurus',
  url: 'https://docusaurus.io',
  // your site config ...
};

export default config;
```

```js title="docusaurus.config.js"
export default function configCreator() {
  return {
    title: 'Docusaurus',
    url: 'https://docusaurus.io',
    // your site config ...
  };
}
```

```js title="docusaurus.config.js"
export default async function createConfigAsync() {
  return {
    title: 'Docusaurus',
    url: 'https://docusaurus.io',
    // your site config ...
  };
}
```

:::tip[使用僅限 ESM 的套件]

使用非同步配置創建函數有助於導入僅限 ESM 的模組（特別是大多數 Remark 插件）。通過動態導入可以實現此類模組的導入：

```js title="docusaurus.config.js"
export default async function createConfigAsync() {
  // 使用動態導入替代 require('esm-lib')
  // highlight-next-line
  const lib = await import('lib');

  return {
    title: 'Docusaurus',
    url: 'https://docusaurus.io',
    // 其餘網站配置...
  };
}
```

:::

## `docusaurus.config.js` 應包含哪些內容？ {#what-goes-into-a-docusaurusconfigjs}

即使您正在開發網站，也無需從頭編寫 `docusaurus.config.js`。所有模板都附帶包含常用選項默認值的 `docusaurus.config.js`。

但了解配置的設計與實現原理仍對您有所幫助。

Docusaurus 配置的高層級概述可分為：

<TOCInline toc={toc} minHeadingLevel={3} maxHeadingLevel={3} />

### 網站元數據 {#site-metadata}

網站元數據包含重要的全局元數據，如 `title`、`url`、`baseUrl` 和 `favicon`。

這些數據會在多處使用，例如網站標題和標頭、瀏覽器標籤圖標、社交分享（Facebook、X）信息，甚至用於生成提供靜態文件的正確路徑。

### 部署配置 {#deployment-configurations}

部署配置如 `projectName`、`organizationName` 和可選的 `deploymentBranch`，會在您使用 `deploy` 命令部署網站時使用。

建議查閱 [部署文檔](deployment.mdx) 以獲取更多信息。

### 主題、插件和預設配置 {#theme-plugin-and-preset-configurations}

在 `themes`、`plugins` 和 `presets` 欄位中分別列出您網站的[主題](./using-plugins.mdx#using-themes)、[插件](./using-plugins.mdx)和[預設集](./using-plugins.mdx#using-presets)。這些通常是 npm 套件：

```js title="docusaurus.config.js"
export default {
  // ...
  plugins: [
    '@docusaurus/plugin-content-blog',
    '@docusaurus/plugin-content-pages',
  ],
  themes: ['@docusaurus/theme-classic'],
};
```

:::tip

Docusaurus 支援[**模組簡寫**](./using-plugins.mdx#module-shorthands)，可將上述配置簡化為：

```js title="docusaurus.config.js"
export default {
  // ...
  plugins: ['content-blog', 'content-pages'],
  themes: ['classic'],
};
```

:::

也可以從本地目錄載入：

```js title="docusaurus.config.js"
import path from 'path';

export default {
  // ...
  themes: [path.resolve(__dirname, '/path/to/docusaurus-local-theme')],
};
```

若要為插件或主題指定選項，請在配置文件中將插件或主題名稱替換為包含名稱和選項物件的陣列：

```js title="docusaurus.config.js"
export default {
  // ...
  plugins: [
    [
      'content-blog',
      {
        path: 'blog',
        routeBasePath: 'blog',
        include: ['*.md', '*.mdx'],
        // ...
      },
    ],
    'content-pages',
  ],
};
```

若要為預設集中捆綁的插件或主題指定選項，請通過 `presets` 欄位傳遞選項。在此範例中，`docs` 指 `@docusaurus/plugin-content-docs`，`theme` 指 `@docusaurus/theme-classic`。

```js title="docusaurus.config.js"
export default {
  // ...
  presets: [
    [
      '@docusaurus/preset-classic',
      {
        docs: {
          sidebarPath: './sidebars.js',
        },
        theme: {
          customCss: ['./src/css/custom.css'],
        },
      },
    ],
  ],
};
```

:::tip

`presets: [['classic', {...}]]` 簡寫同樣有效。

:::

如需進一步配置主題、插件和預設集的幫助，請參閱[使用插件](./using-plugins.mdx)。

### 自訂配置 {#custom-configurations}

Docusaurus 會防止 `docusaurus.config.js` 出現未知欄位。若要新增自訂欄位，請在 `customFields` 中定義。

範例：

```js title="docusaurus.config.js"
export default {
  // ...
  // highlight-start
  customFields: {
    image: '',
    keywords: [],
  },
  // highlight-end
  // ...
};
```

## 從元件存取配置 {#accessing-configuration-from-components}

您的配置物件將可供網站的所有元件使用。您可以通過 React 上下文以 `siteConfig` 存取它們。

基本範例：

```jsx
import React from 'react';
// highlight-next-line
import useDocusaurusContext from '@docusaurus/useDocusaurusContext';

const Hello = () => {
  // highlight-start
  const {siteConfig} = useDocusaurusContext();
  // highlight-end
  const {title, tagline} = siteConfig;

  return <div>{`${title} · ${tagline}`}</div>;
};
```

:::tip

如果僅需在客戶端使用這些欄位，您可以建立自己的 JS 檔案並將其作為 ES6 模組導入，無需將它們放入 `docusaurus.config.js`。

:::

## 自訂 Babel 配置 {#customizing-babel-configuration}

Docusaurus 預設使用 Babel 轉譯網站的原始碼。若要自訂 Babel 配置，可以在專案根目錄中建立 `babel.config.js` 檔案。

若要使用內建預設集作為基礎配置，請安裝以下套件並使用它：

```bash npm2yarn
npm install --save @docusaurus/babel
```

然後在 `babel.config.js` 檔案中使用該預設集：

```js title="babel.config.js"
export default {
  presets: ['@docusaurus/babel/preset'],
};
```

大多數情況下，預設的預設集配置即可滿足需求。若要自訂 Babel 配置（例如新增 Flow 支援），可以直接編輯此檔案。變更生效需重新啟動 Docusaurus 開發伺服器。
