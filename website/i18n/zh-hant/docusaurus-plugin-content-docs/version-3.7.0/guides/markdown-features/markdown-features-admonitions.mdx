# 提示框

import BrowserWindow from '@site/src/components/BrowserWindow';
import Tabs from '@theme/Tabs';
import TabItem from '@theme/TabItem';
import Admonition from '@theme/Admonition';

除了基本的 Markdown 語法外，我們還提供特殊的提示框語法，只需用三個冒號包裹文字，並在後面加上表示其類型的標籤即可。

範例：

```md
:::note

Some **content** with _Markdown_ `syntax`. Check [this `api`](#).

:::

:::tip

Some **content** with _Markdown_ `syntax`. Check [this `api`](#).

:::

:::info

Some **content** with _Markdown_ `syntax`. Check [this `api`](#).

:::

:::warning

Some **content** with _Markdown_ `syntax`. Check [this `api`](#).

:::

:::danger

Some **content** with _Markdown_ `syntax`. Check [this `api`](#).

:::
```

```mdx-code-block
<BrowserWindow>

:::note

Some **content** with _Markdown_ `syntax`. Check [this `api`](#).

:::

:::tip

Some **content** with _Markdown_ `syntax`. Check [this `api`](#).

:::

:::info

Some **content** with _Markdown_ `syntax`. Check [this `api`](#).

:::

:::warning

Some **content** with _Markdown_ `syntax`. Check [this `api`](#).

:::

:::danger

Some **content** with _Markdown_ `syntax`. Check [this `api`](#).

:::

</BrowserWindow>
```

## 與 Prettier 搭配使用 {#usage-with-prettier}

如果你使用 [Prettier](https://prettier.io) 來格式化你的 Markdown 文件，Prettier 可能會將你的程式碼自動格式化為無效的提示框語法。為避免此問題，請在起始和結束指令周圍添加空行。這也是為什麼我們在此展示的範例都在內容周圍有空行。

```md
<!-- Prettier doesn't change this -->
:::note

Hello world

:::

<!-- Prettier changes this -->
:::note
Hello world
:::

<!-- to this -->
::: note Hello world:::
```

## 指定標題 {#specifying-title}

你也可以選擇性地指定一個標題。

```md
:::note[Your Title **with** some _Markdown_ `syntax`!]

Some **content** with some _Markdown_ `syntax`.

:::
```

```mdx-code-block
<BrowserWindow>

:::note[Your Title **with** some _Markdown_ `syntax`!]

Some **content** with some _Markdown_ `syntax`.

:::

</BrowserWindow>
```

## 嵌套提示框 {#nested-admonitions}

提示框可以嵌套使用。對於每個父級提示框層級，使用更多的冒號 `:`。

```md
:::::info[Parent]

Parent content

::::danger[Child]

Child content

:::tip[Deep Child]

Deep child content

:::

::::

:::::
```

```mdx-code-block
<BrowserWindow>

:::::info[Parent]

Parent content

::::danger[Child]

Child content

:::tip[Deep Child]

Deep child content

:::

::::

:::::

</BrowserWindow>
```

## 與 MDX 搭配使用 {#admonitions-with-mdx}

你也可以在提示框中使用 MDX！

```jsx
import Tabs from '@theme/Tabs';

import TabItem from '@theme/TabItem';

:::tip[Use tabs in admonitions]

<Tabs>
  <TabItem value="apple" label="Apple">This is an apple 🍎</TabItem>
  <TabItem value="orange" label="Orange">This is an orange 🍊</TabItem>
  <TabItem value="banana" label="Banana">This is a banana 🍌</TabItem>
</Tabs>

:::
```

## 在 JSX 中使用 {#usage-in-jsx}

在 Markdown 之外，你可以使用 `@theme/Admonition` 組件來獲得相同的輸出。

```jsx title="MyReactPage.jsx"
import Admonition from '@theme/Admonition';

export default function MyReactPage() {
  return (
    <div>
      <Admonition type="info">
        <p>Some information</p>
      </Admonition>
    </div>
  );
}
```

接受的類型與上述相同：`note`、`tip`、`danger`、`info`、`warning`。你可以選擇性地通過傳遞一個 JSX 元素或字符串來指定圖標，或者指定一個標題：

```jsx title="MyReactPage.jsx"
<Admonition type="tip" icon="💡" title="Did you know...">
  Use plugins to introduce shorter syntax for the most commonly used JSX
  elements in your project.
</Admonition>
```

```mdx-code-block
<BrowserWindow>
  <Admonition type="tip" icon="💡" title="Did you know...">
    Use plugins to introduce shorter syntax for the most commonly used JSX
    elements in your project.
  </Admonition>
</BrowserWindow>
```

## 自訂提示框 {#customizing-admonitions}

提示框有兩種自訂方式：**解析**和**渲染**。

### 自訂渲染行為 {#customizing-rendering-behavior}

你可以通過 [swizzling](../../swizzling.mdx) 來自訂每個提示框類型的渲染方式。通常可以通過簡單的包裝來實現你的目標。例如，在以下範例中，我們僅替換了 `info` 提示框的圖標。

```jsx title="src/theme/Admonition.js"
import React from 'react';
import Admonition from '@theme-original/Admonition';
import MyCustomNoteIcon from '@site/static/img/info.svg';

export default function AdmonitionWrapper(props) {
  if (props.type !== 'info') {
    return <Admonition title="My Custom Admonition Title" {...props} />;
  }
  return <Admonition icon={<MyCustomNoteIcon />} {...props} />;
}
```

### 自訂解析行為 {#customizing-parsing-behavior}

提示框是通過 [Remark 插件](./markdown-features-plugins.mdx) 實現的。該插件設計為可配置。要為特定內容插件（文檔、博客、頁面）自訂 Remark 插件，請通過 `admonitions` 鍵傳遞選項。

```js title="docusaurus.config.js"
export default {
  presets: [
    [
      '@docusaurus/preset-classic',
      {
        docs: {
          admonitions: {
            keywords: ['note', 'tip', 'info', 'warning', 'danger'],
            extendDefaults: true,
          },
        },
      },
    ],
  ],
};
```

該插件接受以下選項：

- `keywords`: 可用作提示框類型的關鍵字數組。
- `extendDefaults`: 提供的選項（如 `keywords`）是否應合併到現有默認值中。默認為 `true`。

`keyword` 將作為 `Admonition` 組件的 `type` 屬性傳遞。

### 自訂提示框類型組件 {#custom-admonition-type-components}

默認情況下，主題不知道如何處理自訂的提示框關鍵字，例如 `:::my-custom-admonition`。你需要將每個提示框關鍵字映射到一個 React 組件，以便主題知道如何渲染它們。

如果您透過以下配置註冊了一個新的 admonition 類型 `my-custom-admonition`：

```js title="docusaurus.config.js"
export default {
  // ...
  presets: [
    [
      'classic',
      {
        // ...
        docs: {
          admonitions: {
            keywords: ['my-custom-admonition'],
            extendDefaults: true,
          },
        },
      },
    ],
  ],
};
```

您可以為 `:::my-custom-admonition` 提供對應的 React 元件，方法是建立以下檔案（由於它不是 React 元件檔案，因此無法透過 swizzling 取得）：

```js title="src/theme/Admonition/Types.js"
import React from 'react';
import DefaultAdmonitionTypes from '@theme-original/Admonition/Types';

function MyCustomAdmonition(props) {
  return (
    <div style={{border: 'solid red', padding: 10}}>
      <h5 style={{color: 'blue', fontSize: 30}}>{props.title}</h5>
      <div>{props.children}</div>
    </div>
  );
}

const AdmonitionTypes = {
  ...DefaultAdmonitionTypes,

  // Add all your custom admonition types here...
  // You can also override the default ones if you want
  'my-custom-admonition': MyCustomAdmonition,
};

export default AdmonitionTypes;
```

現在您可以在 Markdown 檔案中使用新的 admonition 關鍵字，它將透過您的自訂邏輯進行解析和渲染：

```md
:::my-custom-admonition[My Title]

It works!

:::
```

<BrowserWindow>

:::my-custom-admonition[My Title]

It works!

:::

</BrowserWindow>
