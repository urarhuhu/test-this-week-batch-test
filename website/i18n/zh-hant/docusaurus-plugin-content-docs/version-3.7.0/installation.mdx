# 安裝


Docusaurus 由一組 npm [套件](https://github.com/facebook/docusaurus/tree/main/packages)組成。

:::tip

使用 **[快速入門](introduction.mdx#fast-track)** 在 **5 分鐘 ⏱** 內了解 Docusaurus！

透過 **[docusaurus.new](https://docusaurus.new)** 直接在瀏覽器中測試 Docusaurus！

:::

## 需求 {#requirements}

- [Node.js](https://nodejs.org/en/download/) 版本 18.0 或更高（可透過執行 `node -v` 檢查）。您可以使用 [nvm](https://github.com/nvm-sh/nvm) 來管理單一機器上的多個 Node 版本。
  - 安裝 Node.js 時，建議勾選所有與依賴項相關的選項。

## 建立專案網站骨架 {#scaffold-project-website}

安裝 Docusaurus 最簡單的方法是使用 [`create-docusaurus`](./api/misc/create-docusaurus.mdx) 命令列工具，它能幫助您建立一個 Docusaurus 網站的基本骨架。您可以在新的空儲存庫或現有儲存庫中的任何位置執行此命令，它將建立一個包含骨架檔案的新目錄。

```bash
npx create-docusaurus@latest my-website classic
```

我們推薦使用 `classic` 模板，以便快速開始，它包含 Docusaurus 1 中的功能。`classic` 模板包含 `@docusaurus/preset-classic`，其中包括標準文件、部落格、自訂頁面和 CSS 框架（支援深色模式）。使用經典模板可以極快地啟動並運行，並在您對 Docusaurus 更加熟悉後進行自訂。

您也可以透過傳遞 `--typescript` 標誌來使用模板的 TypeScript 變體。詳見 [TypeScript 支援](./typescript-support.mdx) 以獲取更多資訊。

```bash
npx create-docusaurus@latest my-website classic --typescript
```

:::info[Meta-Only]

如果您正在為 Meta 開源專案建立新的 Docusaurus 網站，請在內部儲存庫中執行此命令，它會帶有一些有用的 Meta 特定預設值：

```bash
scarf static-docs-bootstrap
```

:::

<details>
  <summary>Alternative installation commands</summary>

You can also initialize a new project using your preferred project manager:

```bash npm2yarn
npm init docusaurus
```

</details>

執行 `npx create-docusaurus@latest --help`，或查看其 [API 文件](./api/misc/create-docusaurus.mdx) 以獲取有關所有可用標誌的更多資訊。

## 專案結構 {#project-structure}

假設您選擇了經典模板並將您的網站命名為 `my-website`，您將在 `my-website/` 目錄下看到以下生成的檔案：

```bash
my-website
├── blog
│   ├── 2019-05-28-hola.md
│   ├── 2019-05-29-hello-world.md
│   └── 2020-05-30-welcome.md
├── docs
│   ├── doc1.md
│   ├── doc2.md
│   ├── doc3.md
│   └── mdx.md
├── src
│   ├── css
│   │   └── custom.css
│   └── pages
│       ├── styles.module.css
│       └── index.js
├── static
│   └── img
├── docusaurus.config.js
├── package.json
├── README.md
├── sidebars.js
└── yarn.lock
```

### 專案結構概述 {#project-structure-rundown}

- `/blog/` - 存放部落格的 Markdown 檔案。若您已停用部落格外掛程式，可刪除此目錄；或透過設定 `path` 選項來變更目錄名稱。詳情請參閱[部落格指南](blog.mdx)
- `/docs/` - 存放文件的 Markdown 檔案。可在 `sidebars.js` 中自訂文件側邊欄的排序。若您已停用文件外掛程式，可刪除此目錄；或透過設定 `path` 選項來變更目錄名稱。詳情請參閱[文件指南](./guides/docs/docs-introduction.mdx)
- `/src/` - 非文件類檔案，例如頁面或自訂 React 元件。您不一定要將非文件檔案集中於此目錄，但統一存放便於後續進行程式碼檢查或處理
  - `/src/pages` - 此目錄下的任何 JSX/TSX/MDX 檔案都會轉換為網站頁面。詳情請參閱[頁面指南](guides/creating-pages.mdx)
- `/static/` - 靜態資源目錄。此目錄內容將直接複製到最終 `build` 目錄的根路徑下
- `/docusaurus.config.js` - 網站設定檔，功能等同於 Docusaurus v1 的 `siteConfig.js`
- `/package.json` - Docusaurus 網站本質是 React 應用程式，您可自由安裝使用任何 npm 套件
- `/sidebars.js` - 供文件功能使用，用於指定側邊欄中文檔的排序

### 單體式儲存庫 {#monorepos}

若您使用 Docusaurus 為現有專案建立說明文件，單體式儲存庫（monorepo）可能是理想解決方案。此架構能讓多個專案共享相依套件。例如：您的網站可直接使用本地套件來展示最新功能，而無需等待正式版發布。如此一來，貢獻者在開發新功能時就能同步更新文件。以下為單體式儲存庫的目錄結構範例：

```bash
my-monorepo
├── package-a # Another package, your actual project
│   ├── src
│   └── package.json # Package A's dependencies
├── website   # Docusaurus root
│   ├── docs
│   ├── src
│   └── package.json # Docusaurus' dependencies
├── package.json # Monorepo's shared dependencies
```

此情境下，您應在 `./my-monorepo` 目錄內執行 `npx create-docusaurus` 指令。

若使用 Netlify 或 Vercel 等託管服務，需將網站的「基礎目錄」（Base directory）變更為 Docusaurus 的根目錄位置。本例中應設為 `./website`。更多忽略指令設定細節請參閱[部署文件](./deployment.mdx#deploying-to-netlify)。

深入了解單體式儲存庫可查閱 [Yarn 文件](https://yarnpkg.com/features/workspaces)（Yarn 並非唯一解決方案，但屬常見做法），或參考 [Docusaurus](https://github.com/facebook/docusaurus) 與 [Jest](https://github.com/facebook/jest) 的實際案例。

## 啟動開發伺服器 {#running-the-development-server}

要即時預覽檔案編輯效果，可執行本地開發伺服器來運行網站並同步顯示最新變更。

```bash npm2yarn
cd my-website
npm run start
```

預設會自動開啟瀏覽器視窗並載入 [`http://localhost:3000`](http://localhost:3000)。

恭喜！您已成功建立第一個 Docusaurus 網站！請隨意瀏覽站內功能。

## 建置 {#build}

Docusaurus 屬於現代靜態網站生成器，需將網站建置為靜態檔案後部署至網頁伺服器才能瀏覽。執行建置指令：

```bash npm2yarn
npm run build
```

建置完成的檔案將產生於 `/build` 目錄，可部署至任何靜態檔案託管服務如 [GitHub Pages](https://pages.github.com/)、[Vercel](https://vercel.com/) 或 [Netlify](https://www.netlify.com/)。詳情請參閱[部署文件](deployment.mdx)。

## 更新 Docusaurus 版本 {#updating-your-docusaurus-version}

有多種方式可以更新您的 Docusaurus 版本。其中一種可靠的方法是手動修改 `package.json` 中的版本號至所需版本。請注意，所有 `@docusaurus/` 命名空間的套件應使用相同的版本。

import UpgradeGuide from '@site/src/components/UpgradeGuide';

<UpgradeGuide />

接著，在包含 `package.json` 的目錄中，執行您的套件管理器的安裝指令：

```bash npm2yarn
npm install
```

:::tip

`npm install` 可能會報告數個漏洞並建議執行 `npm audit` 來修復。通常這些報告的漏洞（例如 RegExp DOS 漏洞）是無害的，可以安全忽略。也請閱讀這篇文章，它反映了我們的觀點：[npm audit: Broken by Design](https://overreacted.io/npm-audit-broken-by-design/)。

:::

若要確認更新是否成功，請執行：

```bash
npx docusaurus --version
```

您應該會看到正確的版本號輸出。

或者，如果您使用 Yarn，可以執行：

```bash
yarn add @docusaurus/core @docusaurus/preset-classic
```

:::tip

使用 [`@canary` npm 發佈標籤](/community/canary) 來體驗 Docusaurus 尚未正式發佈的新功能。

:::
