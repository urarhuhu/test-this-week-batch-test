# 📦 plugin-debug

```mdx-code-block
import Tabs from '@theme/Tabs';
import TabItem from '@theme/TabItem';
```

除錯外掛程式會在 [`http://localhost:3000/__docusaurus/debug`](http://localhost:3000/__docusaurus/debug) 顯示實用的除錯資訊。

該外掛主要對外掛開發者有用，能更輕鬆檢視 `.docusaurus` 資料夾內容（如建立的路由），同時也能檢查未寫入磁碟的資料結構，例如透過 `contentLoaded` 生命週期載入的外掛資料。

:::info

若透過經典預設集使用此外掛，預設情況下預設集會**在開發環境啟用外掛，在生產環境停用** (`debug: undefined`)，以避免暴露潛在敏感資訊。您可使用 `debug: true` 強制啟用，或 `debug: false` 強制停用。

若使用獨立外掛，可能需要透過檢查環境變數達成相同效果：

```js title="docusaurus.config.js"
export default {
  plugins: [
    // highlight-next-line
    process.env.NODE_ENV === 'production' && '@docusaurus/plugin-debug',
  ].filter(Boolean),
};
```

:::

:::note

若回報錯誤時，我們可能會要求您在生產環境啟用此外掛，以便更輕鬆檢查您的部署設定。

若無任何敏感資訊，您可讓其在生產環境保持啟用[如同我們的做法](/__docusaurus/debug)。

:::

## 安裝 {#installation}

```bash npm2yarn
npm install --save @docusaurus/plugin-debug
```

:::tip

若使用 `@docusaurus/preset-classic` 預設集，則無需將此外掛安裝為依賴項。

您可透過[預設集選項](../../using-plugins.mdx#docusauruspreset-classic)設定此外掛。

:::

## 設定 {#configuration}

此外掛目前無任何選項。

### 設定範例 {#ex-config}

您可透過預設集選項或外掛選項設定此外掛。

:::tip

多數 Docusaurus 使用者透過預設集選項設定此外掛。

:::

```mdx-code-block
<Tabs groupId="api-config-ex">
<TabItem value="preset" label="Preset options">
```

若使用預設集，請透過[預設集選項](../../using-plugins.mdx#docusauruspreset-classic)設定此外掛：

```js title="docusaurus.config.js"
export default {
  presets: [
    [
      '@docusaurus/preset-classic',
      {
        // highlight-next-line
        debug: true, // This will enable the plugin in production
      },
    ],
  ],
};
```

```mdx-code-block
</TabItem>
<TabItem value="plugin" label="Plugin Options">
```

若使用獨立外掛，請直接提供選項給外掛：

```js title="docusaurus.config.js"
export default {
  // highlight-next-line
  plugins: ['@docusaurus/plugin-debug'],
};
```
