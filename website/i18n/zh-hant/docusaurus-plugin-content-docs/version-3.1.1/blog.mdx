# 部落格

import Tabs from '@theme/Tabs';
import TabItem from '@theme/TabItem';

部落格功能讓您能快速部署一個功能完整的部落格。

:::info

查看 [部落格插件 API 參考文件](./api/plugins/plugin-content-blog.mdx) 以獲取完整的選項列表。

:::

## 初始設定 {#initial-setup}

要設定網站的部落格，首先建立一個 `blog` 目錄。

接著，在 `docusaurus.config.js` 中加入部落格的連結項目：

```js title="docusaurus.config.js"
export default {
  themeConfig: {
    // ...
    navbar: {
      items: [
        // ...
        // highlight-next-line
        {to: 'blog', label: 'Blog', position: 'left'}, // or position: 'right'
      ],
    },
  },
};
```

## 新增文章 {#adding-posts}

要在部落格中發布文章，請在部落格目錄內建立一個 Markdown 檔案。

例如，建立一個位於 `website/blog/2019-09-05-hello-docusaurus.md` 的檔案：

```md title="website/blog/2019-09-05-hello-docusaurus.md"
---
title: Welcome Docusaurus
description: This is my first post on Docusaurus.
slug: welcome-docusaurus-v2
authors:
  - name: Joel Marcey
    title: Co-creator of Docusaurus 1
    url: https://github.com/JoelMarcey
    image_url: https://github.com/JoelMarcey.png
  - name: Sébastien Lorber
    title: Docusaurus maintainer
    url: https://sebastienlorber.com
    image_url: https://github.com/slorber.png
tags: [hello, docusaurus-v2]
image: https://i.imgur.com/mErPwqL.png
hide_table_of_contents: false
---

Welcome to this blog. This blog is created with [**Docusaurus 2**](https://docusaurus.io/).

<!-- truncate -->

This is my first post on Docusaurus 2.

A whole bunch of exploration to follow.
```

[front matter](./guides/markdown-features/markdown-features-intro.mdx#front-matter) 可用於為部落格文章添加更多元數據，例如作者資訊，但 Docusaurus 能夠在沒有 front matter 的情況下推斷所有必要的元數據。有關所有可能的欄位，請參閱 [API 文件](api/plugins/plugin-content-blog.mdx#markdown-front-matter)。

## 部落格列表 {#blog-list}

部落格的索引頁面（預設位於 `/blog`）是 _部落格列表頁面_，所有部落格文章都會在此集中顯示。

在部落格文章中使用 `<!--truncate-->` 標記來表示在查看所有已發布部落格文章時將顯示的摘要。`<!--truncate-->` 上方的任何內容都將成為摘要的一部分。請注意，截斷標記上方的部分必須是獨立可渲染的 Markdown。例如：

```md title="website/blog/my-post.md" {7}
---
title: Markdown blog truncation example
---

All these will be part of the blog post summary.

<!-- truncate -->

But anything from here on down will not be.
```

對於使用 `.mdx` 副檔名的檔案，請改用 [MDX](https://mdxjs.com/) 註解 `{/* truncate */}`：

```md title="website/blog/my-post.mdx" {7}
---
title: MDX blog truncation Example
---

All these will be part of the blog post summary.

{/* truncate */}

But anything from here on down will not be.
```

預設情況下，每個部落格列表頁面會顯示 10 篇文章，但您可以透過插件配置中的 `postsPerPage` 選項來控制分頁。如果設定 `postsPerPage: 'ALL'`，則會停用分頁，並在第一頁顯示所有文章。您還可以為部落格列表頁面添加元描述以提升 SEO：

```js title="docusaurus.config.js"
export default {
  // ...
  presets: [
    [
      '@docusaurus/preset-classic',
      {
        blog: {
          // highlight-start
          blogTitle: 'Docusaurus blog!',
          blogDescription: 'A Docusaurus powered blog!',
          postsPerPage: 'ALL',
          // highlight-end
        },
      },
    ],
  ],
};
```

## 部落格側邊欄 {#blog-sidebar}

部落格側邊欄會顯示最近的部落格文章。預設顯示的項目數量為 5，但您可以透過插件配置中的 `blogSidebarCount` 選項進行自訂。設定 `blogSidebarCount: 0` 將完全停用側邊欄，並移除容器。這會增加主容器的寬度。特別地，如果設定 `blogSidebarCount: 'ALL'`，則會顯示 _所有_ 文章。

您還可以透過 `blogSidebarTitle` 選項更改側邊欄的標題文字。例如，如果您設定了 `blogSidebarCount: 'ALL'`，與其使用預設的「Recent posts」，您可能更希望顯示「All posts」：

```js title="docusaurus.config.js"
export default {
  presets: [
    [
      '@docusaurus/preset-classic',
      {
        blog: {
          // highlight-start
          blogSidebarTitle: 'All posts',
          blogSidebarCount: 'ALL',
          // highlight-end
        },
      },
    ],
  ],
};
```

## 部落格文章日期 {#blog-post-date}

Docusaurus 會從多種模式中提取 `YYYY-MM-DD` 日期，例如 `YYYY-MM-DD-my-blog-post-title.md` 或 `YYYY/MM/DD/my-blog-post-title.md`。這讓您可以輕鬆地按年、按月分組部落格文章，或使用扁平結構。

<details>
<summary>Supported date extraction patterns</summary>

| Pattern | Example |
| --- | --- |
| Single file | `2021-05-28-my-blog-post-title.md` |
| MDX file | `2021-05-28-my-blog-post-title.mdx` |
| Single folder + `index.md` | `2021-05-28-my-blog-post-title/index.md` |
| Folder named by date | `2021-05-28/my-blog-post-title.md` |
| Nested folders by date | `2021/05/28/my-blog-post-title.md` |
| Partially nested folders by date | `2021/05-28-my-blog-post-title.md` |
| Nested folders + `index.md` | `2021/05/28/my-blog-post-title/index.md` |
| Date in the middle of path | `category/2021/05-28-my-blog-post-title.md` |

Docusaurus can extract the date from the posts using any of the naming patterns above. It is advisable to choose one pattern and apply it to all posts to avoid confusion.

</details>

:::tip

使用資料夾可以方便地將部落格文章的圖片與 Markdown 檔案放在一起。

:::

此命名規則為可選項，您亦可透過 front matter 提供日期。由於 front matter 遵循支援日期時間標記法的 YAML 語法，若需更精確的發布時間，可採用 front matter。例如同一日發布多篇文章時，可依當日時間排序：

```md title="earlier-post.md"
---
date: 2021-09-13T10:00
---
```

```md title="later-post.md"
---
date: 2021-09-13T18:00
---
```

## 部落格文章作者 {#blog-post-authors}

使用 `authors` front matter 欄位來宣告部落格文章作者。作者至少需具備 `name` 或 `image_url` 屬性。Docusaurus 會運用 `url`、`email` 和 `title` 等資訊，但其他任何資訊皆可保留。

### 全域作者 {#global-authors}

對固定發文的作者而言，在每篇部落格文章中維護行內作者資訊可能相當繁瑣。

```yml title="website/blog/authors.yml"

  image_url: https://github.com/JoelMarcey.png

  name: Sébastien Lorber
  title: Docusaurus maintainer
  url: https://sebastienlorber.com
```

## 閱讀時間 {#reading-time}

Docusaurus 會根據字數為每篇部落格文章生成閱讀時間估算。我們提供選項來自訂此功能。

```js title="docusaurus.config.js"
export default {
  presets: [
    [
      '@docusaurus/preset-classic',
      {
        blog: {
          // highlight-start
          showReadingTime: true, // When set to false, the "x min read" won't be shown
          readingTime: ({content, frontMatter, defaultReadingTime}) =>
            defaultReadingTime({content, options: {wordsPerMinute: 300}}),
          // highlight-end
        },
      },
    ],
  ],
};
```

`readingTime` 回調函數接收三個參數：部落格內容文字（字串形式）、front matter（字串鍵值對的記錄）以及預設的閱讀時間計算函數。它會返回一個數字（閱讀時間分鐘數）或 `undefined`（停用該頁面的閱讀時間顯示）。

預設的閱讀時間計算可接受額外選項：`wordsPerMinute`（數字，預設值為 300）和 `wordBound`（從字串到布林值的函數）。若傳入 `wordBound` 的字串應被視為單詞邊界（預設為空格、製表符和換行符），則該函數應返回 `true`。

:::tip

使用回調函數滿足所有自訂需求：

```mdx-code-block

```

**停用單一頁面的閱讀時間：**




## 訂閱源 {#feed}

您可以通過設定 `feedOptions` 來生成 RSS/Atom/JSON 訂閱源。預設情況下會生成 RSS 和 Atom 訂閱源。若要停用訂閱源生成，請將 `feedOptions.type` 設為 `null`。

```ts
type FeedType = 'rss' | 'atom' | 'json';

type BlogOptions = {
  feedOptions?: {
    type?: FeedType | 'all' | FeedType[] | null;
    title?: string;
    description?: string;
    copyright: string;
    language?: string; // possible values: http://www.w3.org/TR/REC-html40/struct/dirlang.html#langcodes
    limit?: number | false | null; // defaults to 20
    /** Allow control over the construction of BlogFeedItems */
    createFeedItems?: (params: {
      blogPosts: BlogPost[];
      siteConfig: DocusaurusConfig;
      outDir: string;
      defaultCreateFeedItems: (params: {
        blogPosts: BlogPost[];
        siteConfig: DocusaurusConfig;
        outDir: string;
      }) => Promise<BlogFeedItem[]>;
    }) => Promise<BlogFeedItem[]>;
  };
};
```

使用範例：

```js title="docusaurus.config.js"
export default {
  // ...
  presets: [
    [
      '@docusaurus/preset-classic',
      {
        blog: {
          // highlight-start
          feedOptions: {
            type: 'all',
            copyright: `Copyright © ${new Date().getFullYear()} Facebook, Inc.`,
            createFeedItems: async (params) => {
              const {blogPosts, defaultCreateFeedItems, ...rest} = params;
              return defaultCreateFeedItems({
                // keep only the 10 most recent blog posts in the feed
                blogPosts: blogPosts.filter((item, index) => index < 10),
                ...rest,
              });
            },
          },
          // highlight-end
        },
      },
    ],
  ],
};
```

訂閱源可在此路徑找到：

<TabItem value="RSS">

```text

```

</TabItem>
<TabItem value="Atom">

```text
https://example.com/blog/atom.xml
```

</TabItem>
<TabItem value="JSON">

```text

```

</TabItem>

## 進階主題 {#advanced-topics}

### 純部落格模式 {#blog-only-mode}

您可以讓 Docusaurus 網站不設專用首頁，而是將部落格文章列表頁設為索引頁。將 `routeBasePath` 設為 `'/'` 即可透過根路由 `example.com/` 提供部落格內容，而非子路由 `example.com/blog/`。

```js title="docusaurus.config.js"
export default {
  // ...
  presets: [
    [
      '@docusaurus/preset-classic',
      {
        // highlight-next-line
        docs: false, // Optional: disable the docs plugin
        blog: {
          // highlight-next-line
          routeBasePath: '/', // Serve the blog at the site's root
          /* other blog options */
        },
      },
    ],
  ],
};
```

:::warning

請務必刪除現有的首頁 `./src/pages/index.js`，否則會出現兩個檔案映射到相同路由的情況！

:::

:::tip

對於僅需使用文檔功能的用戶，亦有「純文檔模式」可供選擇。詳見[純文檔模式](./guides/docs/docs-introduction.mdx)獲取詳細配置指南或關於`routeBasePath`的進階說明。

:::

### 多博客系統 {#multiple-blogs}

經典主題預設單一博客架構，因此僅包含單一博客插件實例。若需在同一網站部署多個博客，可通過在`docusaurus.config.js`的`plugins`選項中配置額外博客插件實現。

將`routeBasePath`設為第二博客的目標URL路徑。請注意此路徑必須與首個博客不同，以避免路徑衝突！同時需將`path`指向存放第二博客條目的目錄路徑。

如[多實例插件](./using-plugins.mdx#multi-instance-plugins-and-plugin-ids)文檔所述，必須為各插件分配唯一ID。

```js title="docusaurus.config.js"
export default {
  // ...
  plugins: [
    [
      '@docusaurus/plugin-content-blog',
      {
        /**
         * Required for any multi-instance plugin
         */
        id: 'second-blog',
        /**
         * URL route for the blog section of your site.
         * *DO NOT* include a trailing slash.
         */
        routeBasePath: 'my-second-blog',
        /**
         * Path to data on filesystem relative to site dir.
         */
        path: './my-second-blog',
      },
    ],
  ],
};
```

實例可參考我們部署的第二博客[此處](/tests/blog)。
