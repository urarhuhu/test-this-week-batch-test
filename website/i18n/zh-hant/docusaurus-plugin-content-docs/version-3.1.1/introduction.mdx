# 介紹

⚡️ Docusaurus 能幫助您快速打造出**精美的文件網站**。

💸 自建技術堆疊成本高昂。不如**專注於內容**，只需撰寫 Markdown 檔案即可。

💥 準備好進階功能了嗎？使用**版本控制、國際化、搜尋和主題客製化**等進階功能。

💅 查看**[最佳 Docusaurus 網站範例](/showcase?tags=favorite)**獲取靈感，並閱讀一些**[用戶見證](https://x.com/sebastienlorber/timelines/1392048416872706049)**。

🧐 Docusaurus 是**靜態網站生成器**。它建構出**單頁應用程式**，具備快速的客戶端導航，充分發揮**React**的互動性優勢。它提供開箱即用的**文件功能**，但也能用於建立**任何類型的網站**（個人網站、產品、部落格、行銷登陸頁等）。

![](/img/slash-introducing.svg)

## 快速入門 ⏱️ {#fast-track}

透過實作，**5分鐘**快速理解 Docusaurus！

建立新的 Docusaurus 網站並遵循**極簡**內建教學。

安裝 [Node.js](https://nodejs.org/en/download/) 並建立新 Docusaurus 網站：

```bash
npx create-docusaurus@latest my-website classic
```

啟動網站：

```bash
cd my-website
npx docusaurus start
```

開啟 [`http://localhost:3000`](http://localhost:3000) 並跟隨教學。

:::tip

使用 **[docusaurus.new](https://docusaurus.new)** 立即在瀏覽器中測試 Docusaurus！

或線上閱讀 **[5分鐘教學](https://tutorial.docusaurus.io)**。

:::

## 從 v1 遷移 {#migrating-from-v1}

Docusaurus v2+ 是對 v1 的全面重寫，採用了完全現代化的工具鏈。在 [v2 正式發布](https://docusaurus.io/blog/2022/08/01/announcing-docusaurus-2.0) 後，我們強烈建議您**使用 Docusaurus v2+ 而非 v1**，因 v1 已停止維護。

已有 [眾多用戶](/showcase) 採用 Docusaurus v2+（[趨勢分析](https://www.npmtrends.com/docusaurus-vs-@docusaurus/core)）。

**以下情況請使用 Docusaurus v2+：**

- :white_check_mark: 需要現代 Jamstack 文件網站
- :white_check_mark: 需要具備客戶端路由的單頁應用程式 (SPA)
- :white_check_mark: 需要完整 React 與 MDX 功能
- :white_check_mark: 不需支援 IE11

**以下情況請使用 [Docusaurus v1](https://v1.docusaurus.io/)：**

- :x: 不需要單頁應用程式 (SPA)
- :x: 需支援 IE11（...真的需要嗎？IE [已終止支援](https://docs.microsoft.com/en-us/lifecycle/products/internet-explorer-11)）

現有 v1 用戶若想升級至 v2+，可參閱我們的[遷移指南](./migration/index.mdx)。

## 功能特色 {#features}

Docusaurus 的設計高度重視開發者與貢獻者的使用體驗。

- ⚛️ **以 React 與 💚 打造**：
  - 透過 React 進行擴展與客製化
  - 提供自訂 React 元件，完整掌控網站的瀏覽體驗
- **可插拔架構**：
  - 使用基礎模板快速建立網站，再透過進階功能與插件強化
  - 將插件開源供社群共享
- ✂️ **開發者體驗**：
  - 立即開始撰寫文件
  - 通用配置入口提升協作者維護性
  - 變更時支援熱重載與閃電般增量建置
  - 基於路由的程式碼與資料分割
  - 輕鬆部署至 GitHub Pages、Netlify、Vercel 等服務

我們的共同目標——幫助使用者快速找到所需內容並更好理解您的產品。我們分享最佳實踐，助您打造優質文件網站。

- 🎯 **SEO 友善**：
  - 為所有可能路徑靜態生成 HTML 檔案
  - 頁面專屬 SEO 優化，讓使用者直接找到相關問題的官方文件
- 📝 **基於 MDX**：
  - 透過嵌入 Markdown 的 JSX 與 React 編寫互動元件
  - 透過即時編輯器分享程式碼，讓使用者當場愛上您的產品
- 🔍 **搜尋功能**：全站內容皆可搜尋
- 💾 **文件版本控制**：保持文件與專案版本同步
- 🌍 **國際化 (i18n)**：支援多語言站點翻譯

Docusaurus v2+ 天生具備極致效能，並貼心照顧所有使用者的可訪問性需求。

- ⚡️ **閃電速度**。遵循 [PRPL 模式](https://developers.google.com/web/fundamentals/performance/prpl-pattern/)，確保內容極速載入
- 🦖 **無障礙設計**。注重可訪問性，讓所有使用者平等獲取內容

## 設計原則 {#design-principles}

- **低學習曲線**：Docusaurus API 簡潔易學。即使需要更多程式碼與時間，使用者仍能實現大多數功能。寧缺毋濫的抽象設計，避免使用者被迫繞過錯誤抽象層。必看演講—[最小化 API 表面積](https://www.youtube.com/watch?v=4anAwXYqLG8)
- **直覺化**：查看專案目錄或新增功能時不會感到壓力，採用熟悉的方式直觀擴展
- **分層架構**：內容/主題/樣式各層關注點分離，模組化抽象清晰
- **合理預設**：為常見場景預設效能優化與配置，同時保留覆寫選項
- **無廠商鎖定**：不強制使用預設插件或 CSS（但強烈建議）。僅核心基礎設施（如 React Loadable 和 React Router）不可替換以維持預設效能優化。Markdown 引擎、CSS 框架、方法論等高層架構完全由使用者決定

我們相信，了解工具運作原理能提升開發者的使用能力。因此我們詳細說明架構與各元件，幫助使用者深入理解並更熟練運用。

## 與其他工具比較 {#comparison-with-other-tools}

在所有靜態網站生成器中，Docusaurus 專注於文件網站並提供大量開箱即用功能。

我們也研究其他主流靜態網站生成器，分享比較見解，協助您在眾多選擇中找到合適方案。

### Gatsby {#gatsby}

[Gatsby](https://www.gatsbyjs.com/) 具備豐富功能，擁有龐大的插件生態系統，能實現 Docusaurus 的所有功能。但這也意味著更高的學習曲線。Gatsby 擅長多種應用場景，適合構建各類網站。而 Docusaurus 則專注於一件事——成為撰寫與發布內容的最佳工具。

GraphQL 是 Gatsby 的核心技術之一，但構建 Gatsby 網站並非必須使用 GraphQL。在大多數靜態網站場景中，您並不需要 GraphQL 提供的靈活性。

Docusaurus v2+ 的許多設計靈感來自 Gatsby 的優秀特性，它是一個出色的替代方案。

[Docz](https://github.com/pedronauck/docz) 是基於 Gatsby 的文檔網站主題，目前功能不如 Docusaurus 完善。

### Next.js {#nextjs}

[Next.js](https://nextjs.org/) 是另一個熱門的混合式 React 框架。雖然可用於構建文檔網站，但缺乏對文檔場景的專門優化，實現 Docusaurus 開箱即用的功能需要更多開發工作。

[Nextra](https://github.com/shuding/nextra) 是基於 Next.js 的靜態網站生成器，目前功能不如 Docusaurus 全面。

### VuePress {#vuepress}

[VuePress](https://vuepress.vuejs.org/) 與 Docusaurus 有諸多相似之處——兩者都專注於內容型網站並提供開箱即用的文檔功能。不同之處在於 VuePress 基於 Vue 構建，而 Docusaurus 基於 React。如需 Vue 技術棧的解決方案，VuePress 是不錯的選擇。

### MkDocs {#mkdocs}

[MkDocs](https://www.mkdocs.org/) 是流行的 Python 靜態網站生成器，核心價值主張與 Docusaurus 相似。

如果您不需要單頁應用且不計劃使用 React，這是個理想的選擇。

[Material for MkDocs](https://squidfunk.github.io/mkdocs-material/) 提供了精美的主題方案。

### Docsify {#docsify}

[Docsify](https://docsify.js.org/) 能快速創建文檔網站，但並非靜態網站生成器且不利於 SEO 優化。

### GitBook {#gitbook}

[GitBook](https://www.gitbook.com/) 擁有簡潔的設計，曾被眾多開源項目採用。隨著其轉向商業化產品定位，許多功能已不再符合開源文檔站點的需求，促使諸多項目遷移。您可閱讀 [Redux 遷移至 Docusaurus 的討論](https://github.com/reduxjs/redux/issues/3161) 了解詳情。

當前 GitBook 僅對開源和非營利團隊免費，而 Docusaurus 對所有用戶免費開放。

### Jekyll {#jekyll}

[Jekyll](https://github.com/jekyll/jekyll) 是最成熟的靜態網站生成器之一——事實上在 Docusaurus 出現前，Facebook 多數開源網站都基於 Jekyll 構建！它的入門門檻極低，我們希望延續這種簡潔的開發體驗。

與通過靜態 HTML 和 `<script /> 標籤添加交互性的傳統方案不同，Docusaurus 網站是 React 應用程序。借助現代 JavaScript 生態工具鏈，我們致力於在文檔站點性能、資源構建流程優化和易用性方面樹立新標準。
