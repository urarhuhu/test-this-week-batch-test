## 語法高亮 {#syntax-highlighting}

程式碼區塊是由三個反引號包圍的文字區塊。您可查閱 [MDX 規範](https://github.com/mdx-js/specification)了解詳細標準。

````md
```js
console.log('Every repo must come with a mascot.');
```
````

使用對應的語言元字串標記程式碼區塊，Docusaurus 將自動啟用 [Prism React Renderer](https://github.com/FormidableLabs/prism-react-renderer) 提供的語法高亮功能。

### 主題設定 {#theming}

預設採用 Prism 的 [Palenight](https://github.com/FormidableLabs/prism-react-renderer/blob/master/packages/prism-react-renderer/src/themes/palenight.ts) 語法高亮主題。您可透過修改 docusaurus.config.js 中 `themeConfig` 的 `prism.theme` 字段切換其他主題。

例如，若想改用 `dracula` 高亮主題：

```js title="docusaurus.config.js"
module.exports = {
  themeConfig: {
    prism: {
      // highlight-next-line
      theme: require('prism-react-renderer/themes/dracula'),
    },
  },
};
```

由於 Prism 主題本質是 JS 物件，您也可自行編寫主題。Docusaurus 強化了 `github` 和 `vsDark` 主題以提供更豐富的高亮效果，可參考我們的 [淺色](https://github.com/facebook/docusaurus/blob/main/website/src/utils/prismLight.ts) 與 [深色](https://github.com/facebook/docusaurus/blob/main/website/src/utils/prismDark.ts) 主題實作。

### 支援語言 {#supported-languages}

Docusaurus 預設內建 [常用語言子集](https://github.com/FormidableLabs/prism-react-renderer/blob/master/packages/generate-prism-languages/index.ts#L9-L23)。

:::warning

部分熱門語言如 Java、C# 或 PHP 預設未啟用。

:::

若要為其他 [Prism 支援語言](https://prismjs.com/#supported-languages)啟用語法高亮，需在附加語言陣列中定義。

:::note

每個附加語言必須是有效的 Prism 元件名稱。例如 Prism 會將語言標記 `cs` 映射到 `csharp`，但只有 `prism-csharp.js` 作為元件存在，因此需使用 `additionalLanguages: ['csharp']`。您可查閱 `node_modules/prismjs/components` 目錄獲取所有可用元件（語言）。

:::

例如，若要為 PowerShell 語言添加高亮支援：

```js title="docusaurus.config.js"
module.exports = {
  // ...
  themeConfig: {
    prism: {
      // highlight-next-line
      additionalLanguages: ['powershell'],
    },
    // ...
  },
};
```

添加 `additionalLanguages` 後需重新啟動 Docusaurus。

若要為 Prism 尚未支援的語言添加高亮，可替換 `prism-include-languages` 元件：

```bash npm2yarn
npm run swizzle @docusaurus/theme-classic prism-include-languages
```

此操作會在 `src/theme` 目錄生成 `prism-include-languages.js`。您可透過編輯此文件為自訂語言添加高亮支援：

```js title="src/theme/prism-include-languages.js"
const prismIncludeLanguages = (Prism) => {
  // ...

  additionalLanguages.forEach((lang) => {
    require(`prismjs/components/prism-${lang}`);
  });

  // highlight-next-line
  require('/path/to/your/prism-language-definition');

  // ...
};
```

編寫自訂語言定義時，可參考 [Prism 官方語言定義](https://github.com/PrismJS/prism/tree/master/components)。

## 行高亮顯示 {#line-highlighting}

### 使用註解高亮 {#highlighting-with-comments}

您可以使用 `highlight-next-line`、`highlight-start` 和 `highlight-end` 註解來選擇要高亮顯示的行。

````md
```js
function HighlightSomeText(highlight) {
  if (highlight) {
    // highlight-next-line
    return 'This text is highlighted!';
  }

  return 'Nothing highlighted';
}

function HighlightMoreText(highlight) {
  // highlight-start
  if (highlight) {
    return 'This range is highlighted!';
  }
  // highlight-end

  return 'Nothing highlighted';
}
```
````

支援的註解語法：

| Style      | Syntax                   |
| ---------- | ------------------------ |
| C-style    | `/* ... */` and `// ...` |
| JSX-style  | `{/* ... */}`            |
| Bash-style | `# ...`                  |
| HTML-style | `<!-- ... -->`           |

我們會根據程式語言盡可能推斷該使用哪種註解風格，預設允許所有註解風格。如果有目前不支援的註解風格，我們樂意新增！歡迎提交 Pull Request。請注意，不同註解風格沒有語意差異，僅內容有別。

您可以在 `src/css/custom.css` 中為高亮程式碼行設定自訂背景色，以更符合您選擇的語法高亮主題。下方給出的顏色適用於預設高亮主題 (Palenight)，若您使用其他主題，需相應調整顏色。

```css title="/src/css/custom.css"
:root {
  --docusaurus-highlighted-code-line-bg: rgb(72, 77, 91);
}

/* If you have a different syntax highlighting theme for dark mode. */
[data-theme='dark'] {
  /* Color which works with dark mode syntax highlighting theme */
  --docusaurus-highlighted-code-line-bg: rgb(100, 100, 100);
}
```

若您還需要以其他方式樣式化高亮程式碼行，可以針對 `theme-code-block-highlighted-line` CSS 類別進行設定。

### 自訂魔法註解 {#custom-magic-comments}

`// highlight-next-line` 和 `// highlight-start` 等被稱為「魔法註解」，因為它們會被解析並移除，其目的是為下一行或由開始-結束註解對包圍的區段添加元數據。

您可以透過主題配置宣告自訂魔法註解。例如，您可以註冊另一個會添加 `code-block-error-line` 類別名稱的魔法註解：

```mdx-code-block
<Tabs>
<TabItem value="docusaurus.config.js">
```

```js
module.exports = {
  themeConfig: {
    prism: {
      magicComments: [
        // Remember to extend the default highlight class name as well!
        {
          className: 'theme-code-block-highlighted-line',
          line: 'highlight-next-line',
          block: {start: 'highlight-start', end: 'highlight-end'},
        },
        // highlight-start
        {
          className: 'code-block-error-line',
          line: 'This will error',
        },
        // highlight-end
      ],
    },
  },
};
```

若您在元數據字串中使用數字範圍（`{1,3-4}` 語法），Docusaurus 會套用第一個 `magicComments` 條目的類別名稱。預設情況下，這是 `theme-code-block-highlighted-line`，但若您變更 `magicComments` 配置並使用不同的條目作為第一個，元數據字串範圍的意義也會隨之改變。

你可以透過設定 `magicComments: []` 來停用預設的行高亮註解。如果沒有設定任何魔法註解配置，但 Docusaurus 遇到包含元數據字串範圍的程式碼區塊時，將會報錯——因為沒有類別名稱可供套用（畢竟高亮類別名稱本質上就是一個魔法註解條目）。

每個魔法註解條目都包含三個鍵值：`className`（必填）、`line`（適用於緊接的下一行），或 `block`（包含 `start` 和 `end`，用於標註兩個註解之間的整個區塊）。

透過 CSS 鎖定類別已能實現許多效果，但你還可以透過[元件替換](../../swizzling.mdx)徹底釋放此功能的潛力。

```bash npm2yarn
npm run swizzle @docusaurus/theme-classic CodeBlock/Line
```

`Line` 元件會接收類別名稱列表，你可以據此條件式渲染不同的標記結構。

## 互動式程式碼編輯器 {#interactive-code-editor}

（由 [React Live](https://github.com/FormidableLabs/react-live) 驅動）

你可以使用 `@docusaurus/theme-live-codeblock` 外掛建立互動式程式碼編輯器。首先，將此外掛加入你的專案。

```bash npm2yarn
npm install --save @docusaurus/theme-live-codeblock
```

你還需要將此外掛加入 `docusaurus.config.js` 設定檔。

```js {3}
module.exports = {
  // ...
  themes: ['@docusaurus/theme-live-codeblock'],
  // ...
};
```

使用此外掛時，請在語言元數據字串後附加 `live` 來建立程式碼區塊。

````md
```jsx live
function Clock(props) {
  const [date, setDate] = useState(new Date());
  useEffect(() => {
    const timerID = setInterval(() => tick(), 1000);

    return function cleanup() {
      clearInterval(timerID);
    };
  });

  function tick() {
    setDate(new Date());
  }

  return (
    <div>
      <h2>It is {date.toLocaleTimeString()}.</h2>
    </div>
  );
}
```
````

該程式碼區塊將被渲染為互動式編輯器，程式碼的變更會即時反映在結果面板上。

### 命令式渲染 (noInline)

當你的程式碼橫跨多個元件或變數時，應使用 `noInline` 選項來避免錯誤。

## 在程式碼區塊中使用 JSX 標記 {#using-jsx-markup}

Markdown 中的程式碼區塊始終將其內容保留為純文字，這意味著你無法實現如下操作：

```ts
type EditUrlFunction = (params: {
  // This doesn't turn into a link (for good reason!)
  version: <a href="/docs/versioning">Version</a>;
  versionDocsDirPath: string;
  docPath: string;
  permalink: string;
  locale: string;
}) => string | undefined;
```

若需嵌入 HTML 標記（如錨點連結或粗體文字），可使用 `<pre>` 標籤、`<code>` 標籤或 `<CodeBlock>` 元件。

```jsx
<pre>
  <b>Input: </b>1 2 3 4{'\n'}
  <b>Output: </b>"366300745"{'\n'}
</pre>
```

<BrowserWindow>
  <pre>
    <b>{'Input: '}</b>
    {'1 2 3 4\n'}
    <b>{'Output: '}</b>
    {'"366300745"\n'}
  </pre>
</BrowserWindow>

:::warning[MDX 對空白不敏感]

MDX 的行為與 JSX 一致：換行字符即使在 `<pre>` 標籤內也會被轉換為空格。您必須明確寫入換行符才能將其輸出。

:::

:::warning

語法高亮僅適用於純文字字符串。Docusaurus 不會嘗試解析包含 JSX 子元素的代碼塊內容。

:::

## 多語言支援代碼塊 {#multi-language-support-code-blocks}

### Docusaurus npm2yarn 備註插件 {#npm2yarn-remark-plugin}

顯示 npm 和 Yarn 的 CLI 命令是非常常見的需求，例如：

```bash npm2yarn
npm install @docusaurus/remark-plugin-npm2yarn
```

Docusaurus 開箱即用提供此實用功能，免除您每次使用 `Tabs` 組件的麻煩。要啟用此功能，首先按照上述方式安裝 `@docusaurus/remark-plugin-npm2yarn` 套件，然後在 `docusaurus.config.js` 中，為需要此功能的插件（文檔、博客、頁面等），在 `remarkPlugins` 選項中註冊它。（有關配置格式的更多詳情，請參閱[文檔配置](../../api/plugins/plugin-content-docs.mdx#ex-config)）

```js title="docusaurus.config.js"
module.exports = {
  // ...
  presets: [
    [
      '@docusaurus/preset-classic',
      {
        docs: {
          // highlight-start
          remarkPlugins: [
            [require('@docusaurus/remark-plugin-npm2yarn'), {sync: true}],
          ],
          // highlight-end
        },
        pages: {
          // highlight-next-line
          remarkPlugins: [require('@docusaurus/remark-plugin-npm2yarn')],
        },
        blog: {
          // highlight-start
          remarkPlugins: [
            [
              require('@docusaurus/remark-plugin-npm2yarn'),
              {converters: ['pnpm']},
            ],
          ],
          // highlight-end
          // ...
        },
      },
    ],
  ],
};
```

然後通過在代碼塊中添加 `npm2yarn` 鍵來使用它：

````md
```bash npm2yarn
npm install @docusaurus/remark-plugin-npm2yarn
```
````

#### 配置 {#npm2yarn-remark-plugin-configuration}

| Option | Type | Default | Description |
| --- | --- | --- | --- |
| `sync` | `boolean` | `false` | Whether to sync the selected converter across all code blocks. |
| `converters` | `array` | `'yarn'`, `'pnpm'` | The list of converters to use. The order of the converters is important, as the first converter will be used as the default choice. |

## 在 JSX 中使用 {#usage-in-jsx}

在 Markdown 之外，您可以使用 `@theme/CodeBlock` 組件獲得相同的輸出。

```jsx
import CodeBlock from '@theme/CodeBlock';

export default function MyReactPage() {
  return (
    <div>
      {/* highlight-start */}
      <CodeBlock
        language="jsx"
        title="/src/components/HelloCodeTitle.js"
        showLineNumbers>
        {`function HelloCodeTitle(props) {
  return <h1>Hello, {props.name}</h1>;
}`}
      </CodeBlock>
      {/* highlight-end */}
    </div>
  );
}
```

```mdx-code-block
<BrowserWindow>
  <CodeBlock
    language="jsx"
    title="/src/components/HelloCodeTitle.js"
    showLineNumbers>
    {`function HelloCodeTitle(props) {
  return <h1>Hello, {props.name}</h1>;
}`}
  </CodeBlock>
</BrowserWindow>
```

接受的屬性包括 `language`、`title` 和 `showLineNumbers`，與您編寫 Markdown 代碼塊的方式相同。

雖然不鼓勵，但您也可以傳入 `metastring` 屬性，如 `metastring='{1-2} title="/src/components/HelloCodeTitle.js" showLineNumbers'`，這是 Markdown 代碼塊在底層處理的方式。然而，我們建議您[使用註釋來高亮行](#highlighting-with-comments)。

如[先前所述](#using-jsx-markup)，語法高亮僅在子元素為簡單字符串時應用。
