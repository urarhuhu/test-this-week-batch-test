## Requirements {#requirements}

- [Node.js](https://nodejs.org/en/download/) version 16.14 or above (which can be checked by running `node -v`). You can use [nvm](https://github.com/nvm-sh/nvm) for managing multiple Node versions on a single machine installed.
  - When installing Node.js, you are recommended to check all checkboxes related to dependencies.

## Scaffold project website {#scaffold-project-website}

The easiest way to install Docusaurus is to use the command line tool that helps you scaffold a skeleton Docusaurus website. You can run this command anywhere in a new empty repository or within an existing repository, it will create a new directory containing the scaffolded files.

```bash
npx create-docusaurus@latest my-website classic
```

We recommend the `classic` template so that you can get started quickly, and it contains features found in Docusaurus 1. The `classic` template contains `@docusaurus/preset-classic` which includes standard documentation, a blog, custom pages, and a CSS framework (with dark mode support). You can get up and running extremely quickly with the classic template and customize things later on when you have gained more familiarity with Docusaurus.

You can also use the template's TypeScript variant by passing the `--typescript` flag. See [TypeScript support](./typescript-support.mdx) for more information.

```bash
npx create-docusaurus@latest my-website classic --typescript
```

:::info[FB-Only]

If you are setting up a new Docusaurus website for a Facebook open source project, use the `facebook` template instead, which comes with some useful Facebook-specific defaults:

```bash
npx create-docusaurus@latest my-website facebook
```

:::

<details>
  <summary>Alternative installation commands</summary>

You can also initialize a new project using your preferred project manager:

```bash npm2yarn
npm init docusaurus
```

</details>

Run `npx create-docusaurus@latest --help`, or check out its [API docs](./api/misc/create-docusaurus.mdx) for more information about all available flags.

## Project structure {#project-structure}

Assuming you chose the classic template and named your site `my-website`, you will see the following files generated under a new directory `my-website/`:

```bash
my-website
├── blog
│   ├── 2019-05-28-hola.md
│   ├── 2019-05-29-hello-world.md
│   └── 2020-05-30-welcome.md
├── docs
│   ├── doc1.md
│   ├── doc2.md
│   ├── doc3.md
│   └── mdx.md
├── src
│   ├── css
│   │   └── custom.css
│   └── pages
│       ├── styles.module.css
│       └── index.js
├── static
│   └── img
├── docusaurus.config.js
├── package.json
├── README.md
├── sidebars.js
└── yarn.lock
```

### Project structure rundown {#project-structure-rundown}

- `/blog/` - 存放部落格的 Markdown 檔案。若已停用部落格外掛程式可刪除此目錄，或透過設定 `path` 選項變更目錄名稱。詳情請參閱[部落格指南](blog.mdx)
- `/docs/` - 存放文件的 Markdown 檔案。可透過 `sidebars.js` 自訂文件側邊欄排序。若已停用文件外掛程式可刪除此目錄，或透過設定 `path` 選項變更目錄名稱。詳情請參閱[文件指南](./guides/docs/docs-introduction.mdx)
- `/src/` - 非文件類檔案（如頁面或自訂 React 元件）。非文件檔案不強制存放於此，但集中管理便於後續進行程式碼檢查或處理
  - `/src/pages` - 此目錄下的 JSX/TSX/MDX 檔案將自動轉換為網站頁面。詳情請參閱[頁面指南](guides/creating-pages.mdx)
- `/static/` - 靜態資源目錄。此處所有內容將直接複製至最終 `build` 目錄的根路徑
- `/docusaurus.config.js` - 網站組態檔，功能等同於 Docusaurus v1 的 `siteConfig.js`
- `/package.json` - Docusaurus 網站本質是 React 應用程式，可自由安裝使用任何 npm 套件
- `/sidebars.js` - 用於指定文件在側邊欄的顯示順序

### 單體式儲存庫 (Monorepos) {#monorepos}

若需為現有專案建立文件，單體式儲存庫可能是理想解決方案。此架構允許在相似專案間共享依賴項，例如網站可直接使用本地套件展示最新功能（無需依賴已發布版本），貢獻者也能在開發功能時同步更新文件。以下為單體式儲存庫的範例目錄結構：

```bash
my-monorepo
├── package-a # Another package, your actual project
│   ├── src
│   └── package.json # Package A's dependencies
├── website   # Docusaurus root
│   ├── docs
│   ├── src
│   └── package.json # Docusaurus' dependencies
├── package.json # Monorepo's shared dependencies
```

此情境下，應在 `./my-monorepo` 目錄內執行 `npx create-docusaurus`。

若使用 Netlify 或 Vercel 等託管服務，需將網站的「基礎目錄 (Base directory)」變更為 Docusaurus 根目錄位置（本例為 `./website`）。更多忽略指令設定細節請參閱[部署文件](./deployment.mdx#deploying-to-netlify)。

深入了解單體式儲存庫可參閱 [Yarn 文件](https://yarnpkg.com/features/workspaces)（Yarn 非唯一解決方案但屬常見實踐），或參考 [Docusaurus](https://github.com/facebook/docusaurus) 與 [Jest](https://github.com/facebook/jest) 的實際案例。

## 啟動開發伺服器 {#running-the-development-server}

要即時預覽編輯內容，可執行本地開發伺服器來託管網站並同步顯示最新變更。

```bash npm2yarn
cd my-website
npm run start
```

預設將自動開啟瀏覽器視窗並導向 [`http://localhost:3000`](http://localhost:3000)。

恭喜！您已成功建立第一個 Docusaurus 網站！請瀏覽站點內容查看現有功能。

## 建置 {#build}

Docusaurus 屬現代靜態網站生成器，需將網站建置為靜態內容目錄後部署至網頁伺服器才能瀏覽。建置指令如下：

```bash npm2yarn
npm run build
```

建置完成後，所有內容將生成於 `/build` 目錄，可直接部署至 [GitHub Pages](https://pages.github.com/)、[Vercel](https://vercel.com/) 或 [Netlify](https://www.netlify.com/) 等靜態託管服務。詳情請參閱[部署文件](deployment.mdx)。

## 更新 Docusaurus 版本 {#updating-your-docusaurus-version}

There are many ways to update your Docusaurus version. One guaranteed way is to manually change the version number in `package.json` to the desired version. Note that all `@docusaurus/`-namespaced packages should be using the same version.

import UpgradeGuide from '@site/src/components/UpgradeGuide';

<UpgradeGuide />

Then, in the directory containing `package.json`, run your package manager's install command:

```bash npm2yarn
npm install
```

To check that the update occurred successfully, run:

```bash
npx docusaurus --version
```

You should see the correct version as output.

Alternatively, if you are using Yarn, you can do:

```bash
yarn upgrade @docusaurus/core@latest @docusaurus/preset-classic@latest
```

Use new unreleased features of Docusaurus with the [`@canary` npm dist tag](/community/canary)
