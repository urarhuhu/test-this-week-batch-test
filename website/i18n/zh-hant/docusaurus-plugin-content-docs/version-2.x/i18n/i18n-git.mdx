# 國際化 - 使用 Git

一種**可能的翻譯策略**是使用 Git（或任何其他[版本控制系統](https://en.wikipedia.org/wiki/Version_control)）**對翻譯檔案進行版本控制**。

## 權衡取捨 {#tradeoffs}

此策略有以下優點：

- **易於上手**：只需將 `i18n` 資料夾提交到 Git
- **對開發者友好**：Git、GitHub 和拉取請求是主流的開發者工具
- **免費**（或無需額外成本，假設您已使用 Git）
- **低門檻**：無需註冊外部工具
- **有成就感**：貢獻者會因擁有良好的貢獻記錄而感到高興

使用 Git 也存在一些缺點：

- **對非開發者困難**：他們不熟悉 Git 和拉取請求
- **對專業翻譯人員困難**：他們習慣使用 SaaS 翻譯軟體和進階功能
- **難以維護**：您需要保持翻譯檔案與未翻譯檔案**同步**

:::note

一些**大型技術專案**（如 React、Vue.js、MDN、TypeScript、Nuxt.js 等）使用 Git 進行翻譯。

請參考 [Docusaurus 國際化 RFC](https://github.com/facebook/docusaurus/issues/3317) 以了解我們對此類系統的研究筆記和連結。

:::

## 初始化 {#initialization}

以下是如何使用 Git 將新初始化的英文 Docusaurus 網站翻譯成法文的逐步指南，假設您已遵循[國際化教學](./i18n-tutorial.mdx)。

### 準備 Docusaurus 網站 {#prepare-the-docusaurus-site}

初始化一個新的 Docusaurus 網站：

```bash
npx create-docusaurus@latest website classic
```

為法文語言新增網站配置：

```js title="docusaurus.config.js"
module.exports = {
  i18n: {
    defaultLocale: 'en',
    locales: ['en', 'fr'],
  },
  themeConfig: {
    navbar: {
      items: [
        // ...
        {
          type: 'localeDropdown',
          position: 'left',
        },
        // ...
      ],
    },
  },
  // ...
};
```

翻譯首頁：

```jsx title="src/pages/index.js"
import React from 'react';
import Translate from '@docusaurus/Translate';
import Layout from '@theme/Layout';

export default function Home() {
  return (
    <Layout>
      <h1 style={{margin: 20}}>
        <Translate description="The homepage main heading">
          Welcome to my Docusaurus translated site!
        </Translate>
      </h1>
    </Layout>
  );
}
```

### 初始化 `i18n` 資料夾 {#initialize-the-i18n-folder}

使用 [write-translations](../cli.mdx#docusaurus-write-translations-sitedir) CLI 指令初始化法文語系的 JSON 翻譯檔案：

```bash npm2yarn
npm run write-translations -- --locale fr

  1 translations written at i18n/fr/code.json
 11 translations written at i18n/fr/docusaurus-theme-classic/footer.json
  4 translations written at i18n/fr/docusaurus-theme-classic/navbar.json
  3 translations written at i18n/fr/docusaurus-plugin-content-docs/current.json
```

:::tip

使用 `--messagePrefix '(fr) '` 選項讓未翻譯的字串突出顯示。

`Hello` 會顯示為 `(fr) Hello`，這樣可以清楚標示出需要翻譯的部分。

:::

將未翻譯的 Markdown 檔案複製到法文資料夾：

```bash
mkdir -p i18n/fr/docusaurus-plugin-content-docs/current
cp -r docs/** i18n/fr/docusaurus-plugin-content-docs/current

mkdir -p i18n/fr/docusaurus-plugin-content-blog
cp -r blog/** i18n/fr/docusaurus-plugin-content-blog

mkdir -p i18n/fr/docusaurus-plugin-content-pages
cp -r src/pages/**.md i18n/fr/docusaurus-plugin-content-pages
cp -r src/pages/**.mdx i18n/fr/docusaurus-plugin-content-pages
```

將所有這些檔案加入 Git。

### 翻譯檔案 {#translate-the-files}

翻譯 `i18n/fr` 中的 Markdown 和 JSON 檔案，並提交翻譯。

您現在應該能夠以法文啟動網站並看到翻譯內容：

```bash npm2yarn
npm run start -- --locale fr
```

您也可以在本地或 CI 上建置網站：

```bash npm2yarn
npm run build
# or
npm run build -- --locale fr
```

## 維護 {#maintenance}

保持翻譯檔案與原始檔案**一致**可能**具有挑戰性**，尤其是對於 Markdown 文件。

### Markdown 翻譯 {#markdown-translations}

當未翻譯的 Markdown 文件被編輯時，**您有責任維護相應的翻譯檔案**，但目前我們沒有很好的方法來幫助您完成此操作。

To keep your translated sites consistent, when the `website/docs/doc1.md` doc is edited, you need **backport these edits** to `i18n/fr/docusaurus-plugin-content-docs/current/doc1.md`.

### JSON translations {#json-translations}

To help you maintain the JSON translation files, it is possible to run again the [write-translations](../cli.mdx#docusaurus-write-translations-sitedir) CLI command:

```bash npm2yarn
npm run write-translations -- --locale fr
```

New translations will be appended, and existing ones will not be overridden.

:::tip

Reset your translations with the `--override` option.

:::

### Localize edit URLs {#localize-edit-urls}

When the user is browsing a page at `/fr/doc1`, the edit button will link by default to the unlocalized doc at `website/docs/doc1.md`.

Your translations are on Git, and you can use the `editLocalizedFiles: true` option of the docs and blog plugins.

The edit button will link to the localized doc at `i18n/fr/docusaurus-plugin-content-docs/current/doc1.md`.
