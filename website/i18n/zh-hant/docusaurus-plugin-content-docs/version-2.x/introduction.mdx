# 簡介

⚡️ Docusaurus 能協助您快速打造出**美觀的技術文件網站**。

💸 自建技術堆疊成本高昂。請專注於**內容創作**，只需撰寫 Markdown 檔案即可。

💥 需要更多功能？可使用**進階特性**如版本控制、國際化、站內搜索與主題客製化。

💅 查看**[精選 Docusaurus 網站](/showcase?tags=favorite)**獲取靈感，或閱讀**[用戶見證](https://x.com/sebastienlorber/timelines/1392048416872706049)**。

🧐 Docusaurus 是**靜態網站生成器**。它構建具備快速客戶端導航的**單頁應用程式**，充分發揮 **React** 的交互能力。除預置的**技術文件功能**外，亦可用於創建**任何類型網站**（個人網站、產品頁、部落格、行銷登陸頁等）。

![](/img/slash-introducing.svg)

## 快速入門 ⏱️ {#fast-track}

透過實作，**5分鐘**快速理解 Docusaurus！

新建 Docusaurus 專案並跟隨**極簡**內建教學。

安裝 [Node.js](https://nodejs.org/en/download/) 後建立新專案：

```bash
npx create-docusaurus@latest my-website classic
```

啟動開發伺服器：

```bash
cd my-website
npx docusaurus start
```

訪問 [`http://localhost:3000`](http://localhost:3000) 跟隨教學操作。

:::tip

立即透過 **[docusaurus.new](https://docusaurus.new)** 在瀏覽器中體驗 Docusaurus！

或線上閱讀 **[5分鐘教學](https://tutorial.docusaurus.io)**。

:::

## 從 v1 遷移 {#migrating-from-v1}

Docusaurus v2 是基於現代化工具鏈的全面重構版本。自 [v2 正式發布](https://docusaurus.io/blog/2022/08/01/announcing-docusaurus-2.0) 後，我們強烈建議**使用 v2 而非已停止維護的 v1**。

已有 [眾多用戶](/showcase) 採用 Docusaurus v2（[趨勢分析](https://www.npmtrends.com/docusaurus-vs-@docusaurus/core)）。

**適用 v2 的情境：**

- :white_check_mark: 需要現代 Jamstack 技術文件站
- :white_check_mark: 需單頁應用程式 (SPA) 與客戶端路由
- :white_check_mark: 需完整 React 與 MDX 功能
- :white_check_mark: 無需支援 IE11 瀏覽器

**適用 [Docusaurus v1](https://v1.docusaurus.io/) 的情境：**

- :x: 不需單頁應用程式 (SPA)
- :x: 需支援 IE11（注意：IE [已停止支援](https://docs.microsoft.com/en-us/lifecycle/products/internet-explorer-11)）

現有 v1 用戶可參閱 [遷移指南](./migration/migration-overview.mdx) 升級至 v2。

## 功能特色 {#features}

Docusaurus 的設計高度重視開發者與貢獻者的使用體驗。

- ⚛️ **以 React 與 💚 打造**：
  - 透過 React 進行擴展與客製化
  - 提供自訂 React 元件，完整掌控網站的瀏覽體驗
- **模組化設計**：
  - 使用基礎模板快速建立網站，再透過進階功能與插件強化
  - 可開源您的插件與社群共享
- ✂️ **開發者體驗**：
  - 立即開始撰寫文件
  - 通用配置入口提升協作者維護性
  - 變更時支援熱重載與閃電般增量建置
  - 基於路由的程式碼與資料分割
  - 輕鬆部署至 GitHub Pages、Netlify、Vercel 等服務

我們的共同目標——協助使用者快速找到所需內容並更好理解您的產品。我們分享最佳實踐，助您打造優質文件網站。

- 🎯 **SEO 友善**：
  - 為所有可能路徑靜態生成 HTML 檔案
  - 頁面專屬 SEO 優化，讓使用者直接找到相關問題的官方文件
- 📝 **基於 MDX**：
  - 透過嵌入 Markdown 的 JSX 與 React 編寫互動元件
  - 透過即時編輯器分享程式碼，讓使用者當場愛上您的產品
- 🔍 **搜尋功能**：全站內容皆可搜尋
- 💾 **文件版本控制**：保持文件與專案版本同步
- 🌍 **國際化 (i18n)**：支援多語言站點翻譯

Docusaurus 2 天生具備極致效能，並貼心顧及所有使用者的可訪問性。

- ⚡️ **閃電速度**。Docusaurus 2 遵循 [PRPL 模式](https://developers.google.com/web/fundamentals/performance/prpl-pattern/)，確保內容極速載入
- 🦖 **無障礙設計**。注重可訪問性，讓所有使用者平等獲取內容

## 設計原則 {#design-principles}

- **低學習曲線**。Docusaurus API 簡潔易學，即使需要更多程式碼與時間，使用者仍能實現多數功能。寧缺毋濫的抽象設計，避免使用者被迫繞過錯誤抽象層。必看演講—[最小化 API 表面積](https://www.youtube.com/watch?v=4anAwXYqLG8)
- **直覺化**。查看專案目錄或新增功能時不會感到壓力，採用熟悉的方式直觀擴展
- **分層架構**。各層級（內容/主題/樣式）關注點分離清晰，模組化抽象完善
- **合理預設**。為使用者實施常見效能優化與配置，同時保留覆寫選項
- **無廠商鎖定**。不強制使用預設插件或 CSS（但強烈建議）。核心基礎設施如 React Loadable 和 React Router 因預設效能優化不可替換，但高階架構如 Markdown 引擎、CSS 框架等完全由使用者決定

我們相信，了解工具運作原理能提升開發者的使用能力。因此我們致力解說 Docusaurus 的架構與各元件，幫助使用者深入理解並更熟練運用。

## 與其他工具比較 {#comparison-with-other-tools}

在所有靜態網站生成器中，Docusaurus 獨特聚焦於文件網站，並提供眾多開箱即用功能。

我們亦研究其他主流靜態網站生成器，分享比較見解，協助您在眾多選擇中做出決策。

### Gatsby {#gatsby}

[Gatsby](https://www.gatsbyjs.com/) 具備豐富功能與龐大插件生態系，能實現 Docusaurus 的所有功能，但代價是較高的學習曲線。Gatsby 擅長建置各類網站，而 Docusaurus 則專注於單一目標——成為最優秀的技術文檔撰寫與發布工具。

GraphQL 雖是 Gatsby 的核心技術，但建置靜態網站時通常不需其靈活性。Docusaurus 2 的許多設計靈感源自 Gatsby 的優點，使其成為絕佳替代方案。

Docusaurus 2 的諸多設計汲取了 Gatsby 的優點，是極佳的替代選擇。

[Docz](https://github.com/pedronauck/docz) 是基於 Gatsby 的文檔網站主題，目前功能尚不及 Docusaurus 完善。

### Next.js {#nextjs}

[Next.js](https://nextjs.org/) 是熱門的混合式 React 框架，雖能建置文檔網站，但缺乏對文檔場景的專屬優化，需投入大量額外工作才能實現 Docusaurus 的開箱即用功能。

[Nextra](https://github.com/shuding/nextra) 是基於 Next.js 的靜態網站生成器，目前功能完整度低於 Docusaurus。

### VuePress {#vuepress}

[VuePress](https://vuepress.vuejs.org/) 與 Docusaurus 高度相似——皆專注於內容型網站並提供文檔專屬功能，差異在於 VuePress 基於 Vue 框架。若需 Vue 技術棧，VuePress 是合適選擇。

### MkDocs {#mkdocs}

[MkDocs](https://www.mkdocs.org/) 是熱門的 Python 靜態網站生成器，核心價值與 Docusaurus 相近。

若不需要單頁應用且無需 React 技術棧，此為理想選擇。

[Material for MkDocs](https://squidfunk.github.io/mkdocs-material/) 提供精美的主題樣式。

### Docsify {#docsify}

[Docsify](https://docsify.js.org/) 便於快速建立文檔網站，但非靜態網站生成器且不利 SEO 優化。

### GitBook {#gitbook}

[GitBook](https://www.gitbook.com/) 擁有簡潔設計並被眾多開源專案採用。隨著其商業化轉型，現已難以滿足開源文檔需求，促使許多專案遷移（如 [Redux 轉用 Docusaurus 的討論](https://github.com/reduxjs/redux/issues/3161)）。

現階段 GitBook 僅對開源與非營利團隊免費，而 Docusaurus 完全開放給所有使用者。

### Jekyll {#jekyll}

[Jekyll](https://github.com/jekyll/jekyll) 是最成熟的靜態網站生成器之一——事實上在 Docusaurus 之前，Facebook 多數開源網站皆基於 Jekyll 建置！我們致力提供與 Jekyll 相似的簡易開發體驗。

相較於 Jekyll 的靜態 HTML 與 `<script /> 標籤互動模式，Docusaurus 生成的是 React 應用程式。透過現代化 JavaScript 工具鏈，我們期望重新定義文檔網站的效能標準、資源建置流程與優化機制。
