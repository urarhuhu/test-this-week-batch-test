# 簡介

⚡️ Docusaurus 能協助您快速打造出**精美的文件網站**。

💸 自建技術堆疊成本高昂。請專注於**內容創作**，只需撰寫 Markdown 檔案即可。

💥 想進階使用？可運用**進階功能**如版本控制、國際化、搜尋與主題客製化。

💅 查看**[精選 Docusaurus 網站](/showcase?tags=favorite)**獲取靈感。

🧐 Docusaurus 是**靜態網站生成器**。它建構具備快速客戶端導覽的**單頁應用程式**，充分發揮 **React** 的互動性優勢。除內建**文件專屬功能**外，亦可用於建立**任何類型網站**（個人網站、產品頁、部落格、行銷登陸頁等）。

![](/img/slash-introducing.svg)

## 快速入門 ⏱️ {#fast-track}

透過實作，**5分鐘**快速理解 Docusaurus！

新建 Docusaurus 網站並遵循**極簡**嵌入式教學。

安裝 [Node.js](https://nodejs.org/en/download/) 後建立新 Docusaurus 網站：

```bash
npx create-docusaurus@latest my-website classic
```

啟動網站：

```bash
cd my-website
npx docusaurus start
```

開啟 [`http://localhost:3000`](http://localhost:3000) 並跟隨教學指引。

:::tip

立即透過 **[docusaurus.new](https://docusaurus.new)** 在瀏覽器中測試 Docusaurus！

或線上閱讀 **[5分鐘教學](https://tutorial.docusaurus.io)**。

:::

## 從 v1 遷移 {#migrating-from-v1}

Docusaurus v2+ 是基於現代化工具鏈的全面重寫版本。自 [v2 正式發布](https://docusaurus.io/blog/2022/08/01/announcing-docusaurus-2.0) 後，我們強烈建議**使用 v2+ 而非已棄用的 v1 版本**。

已有 [眾多用戶](/showcase) 採用 Docusaurus v2+（[趨勢分析](https://www.npmtrends.com/docusaurus-vs-@docusaurus/core)）。

**適用 Docusaurus v2+ 的情境：**

- :white_check_mark: 需建置現代化 Jamstack 文件網站
- :white_check_mark: 需具備客戶端路由的單頁應用程式 (SPA)
- :white_check_mark: 需完整 React 與 MDX 功能支援
- :white_check_mark: 無需支援 IE11 瀏覽器

**適用 [Docusaurus v1](https://v1.docusaurus.io/) 的情境：**

- :x: 不需單頁應用程式 (SPA)
- :x: 需支援 IE11（注意：IE [已停止官方支援](https://docs.microsoft.com/en-us/lifecycle/products/internet-explorer-11)）

現有 v1 用戶可參閱我們的[遷移指南](./migration/index.mdx)升級至 v2+。

## 功能特色 {#features}

Docusaurus 以高度重視開發者與貢獻者體驗為核心打造。

- ⚛️ **以💚與React建構**：
  - 透過React擴展與自訂
  - 提供自訂React元件，完全掌控網站瀏覽體驗
- 🔌 **可插拔式架構**：
  - 使用基礎模板快速建站，再進階使用功能與插件
  - 開源插件與社群共享
- ✂️ **開發者體驗**：
  - 立即開始撰寫文件
  - 統一配置入口提升協作維護性
  - 熱重載與閃電級增量建置
  - 基於路由的程式碼與資料分割
  - 輕鬆部署至GitHub Pages、Netlify、Vercel等服務

我們的共同目標——幫助使用者快速找到所需，更深入理解您的產品。我們分享最佳實踐，助您打造優質文件站點。

- 🎯 **SEO優化**：
  - 為所有路徑靜態生成HTML文件
  - 頁面專屬SEO，讓使用者直達相關問題的官方文件
- 📝 **基於MDX**：
  - 在Markdown中嵌入JSX與React撰寫互動元件
  - 透過即時編輯器分享程式碼，讓用戶當場愛上您的產品
- 🔍 **全站搜索**：完整內容皆可搜索
- 💾 **文件版本控制**：保持文件與專案版本同步
- 🌍 **國際化(i18n)**：支援多語言站點翻譯

Docusaurus v2+ 天生具備極致效能與全面無障礙訪問能力。

- ⚡️ **閃電速度**。遵循[PRPL模式](https://developers.google.com/web/fundamentals/performance/prpl-pattern/)，確保內容極速載入
- 🦖 **無障礙設計**。注重可訪問性，讓所有用戶平等使用

## 設計原則 {#design-principles}

- **低學習曲線**。API精簡易上手，即使需更多程式碼與時間，用戶仍能實現多數功能。寧缺毋濫的抽象設計，避免用戶被迫繞過不當抽象層。必看演講——[最小API表面積](https://www.youtube.com/watch?v=4anAwXYqLG8)。
- **直覺化**。專案目錄結構與新增功能流程符合直覺，採用熟悉模式降低上手壓力。
- **分層架構**。內容/主題/樣式各層關注點分離，模組化抽象清晰。
- **合理預設**。為常見場景預置效能優化與配置，同時保留覆寫彈性。
- **無廠商鎖定**。雖強烈推薦預設插件與CSS，但皆可替換。僅核心基礎設施(如React Loadable與React Router)因預設優化不可替換，其餘如Markdown引擎、CSS框架等架構選擇完全開放。

我們相信，理解工具運作原理能讓開發者更善用工具。因此我們詳解Docusaurus架構與各組件，助您深入掌握其精髓。

## 與其他工具比較 {#comparison-with-other-tools}

在所有靜態站點生成器中，Docusaurus獨特聚焦文件站點，提供大量開箱即用功能。

我們亦深入研究其他主流靜態站點生成器，分享比較見解，助您在眾多選擇中明智決策。

### Gatsby {#gatsby}

[Gatsby](https://www.gatsbyjs.com/) 具備豐富功能與龐大插件生態系統，能實現 Docusaurus 的所有功能，但代價是較高的學習曲線。Gatsby 擅長多種網站開發，而 Docusaurus 專注於將一件事做到極致——成為編寫與發布內容的最佳工具。

GraphQL 是 Gatsby 的核心技術，但構建靜態網站時通常不需要其靈活性。

Docusaurus v2+ 的許多設計靈感源自 Gatsby 的優點，是極佳的替代方案。

[Docz](https://github.com/pedronauck/docz) 是基於 Gatsby 的文檔網站主題，目前功能不如 Docusaurus 全面。

### Next.js {#nextjs}

[Next.js](https://nextjs.org/) 是熱門的混合式 React 框架，雖能構建文檔網站，但缺乏對文檔場景的預設支持，需投入更多工作量才能實現 Docusaurus 的開箱即用功能。

[Nextra](https://github.com/shuding/nextra) 是基於 Next.js 的靜態網站生成器，目前功能完整度低於 Docusaurus。

### VitePress {#vitepress}

[VitePress](https://vitepress.dev/) 與 Docusaurus 高度相似——均專注內容型網站並提供文檔專屬功能。差異在於 VitePress 基於 Vue，Docusaurus 基於 React。若需 Vue 解決方案，VitePress 是理想選擇。

### MkDocs {#mkdocs}

[MkDocs](https://www.mkdocs.org/) 是熱門的 Python 靜態網站生成器，核心價值與 Docusaurus 相近。

若不需要單頁應用且不計劃使用 React，這是個好選擇。

[Material for MkDocs](https://squidfunk.github.io/mkdocs-material/) 提供精美的主題方案。

### Docsify {#docsify}

[Docsify](https://docsify.js.org/) 易於創建文檔網站，但非靜態網站生成器且不利於 SEO。

### GitBook {#gitbook}

[GitBook](https://www.gitbook.com/) 擁有簡潔設計並被眾多開源項目採用。隨著其轉向商業化產品，許多功能已不符合開源文檔需求，促使項目遷移（如 [Redux 轉用 Docusaurus](https://github.com/reduxjs/redux/issues/3161)）。

現 GitBook 僅對開源與非營利團隊免費，Docusaurus 則完全免費。

### Jekyll {#jekyll}

[Jekyll](https://github.com/jekyll/jekyll) 是最成熟的靜態網站生成器之一（Facebook 開源網站曾廣泛使用）。Docusaurus 旨在提供類似的開發體驗，同時通過現代 JavaScript 工具鏈，在性能、資源構建與優化方面樹立新標準。

相較於 Jekyll 的靜態 HTML 與腳本標籤交互模式，Docusaurus 網站是完整的 React 應用。
