# 支援

Docusaurus 擁有數千名開發者組成的社群。

本頁面列出了您可以參與的 Docusaurus 相關社群；請參閱本節其他頁面以獲取更多線上及線下學習資源。

在參與 Docusaurus 社群之前，[請先閱讀我們的行為準則](https://engineering.fb.com/codeofconduct/)。我們採用了 [貢獻者公約](https://www.contributor-covenant.org/)，並期望所有社群成員遵守其中的準則。

## Stack Overflow {#stack-overflow}

Stack Overflow 是詢問程式碼層級問題或解決特定錯誤的熱門論壇。瀏覽標記為 **docusaurus** 的[現有問題](https://stackoverflow.com/questions/tagged/docusaurus)或[提出您的問題](https://stackoverflow.com/questions/ask?tags=docusaurus)！

## 討論論壇 {#discussion-forums}

有許多線上論壇可供討論最佳實踐、應用程式架構以及 Docusaurus 的未來發展。如果您有可解答的程式碼層級問題，Stack Overflow 通常是更適合的選擇。

- [Docusaurus 線上聊天室](https://discord.gg/docusaurus)
  - [#help-and-questions](https://discord.gg/fwbcrQ3dHR) 用於使用者求助
  - [#contributors](https://discord.gg/6g6ASPA) 用於貢獻者協助
- [Reddit 上的 Docusaurus 社群](https://www.reddit.com/r/docusaurus/)

## 功能請求 {#feature-requests}

對於新功能請求，您可以在我們的[功能請求看板 (Canny)](/feature-requests)上發文。該工具便於路線規劃，並可依投票數排序，讓核心團隊更清楚哪些功能需求較高，相較於較難分類的 GitHub 議題。請避免直接為新功能（尤其是大型功能）提交 Pull Request，因為可能已有人在開發或已列入我們的開發路線。請先與我們討論！

## 新聞 {#news}

有關 Docusaurus 的最新消息，請在 X 上[追蹤 **@docusaurus**](https://x.com/docusaurus) 並關注本網站的[官方 Docusaurus 部落格](/blog)。
